HTML is the skeleton of every web page

HTML (HyperText Markup Language) is the standard language for creating web pages. It describes the structure and meaning of content — not its visual appearance. CSS handles look; JavaScript handles behaviour; HTML defines what everything is.

Every web page your browser loads is, at its core, an HTML document — a plain text file with a .html extension.
HTML uses elements (tags like <p>, <h1>, <a>) to mark up content with meaning.
Browsers parse HTML into a tree called the DOM (Document Object Model) — the live representation CSS and JS operate on.
HTML is not a programming language — it has no logic, loops, or variables. It is a markup language.

Core Conceptual Questions
Click any question to expand — focus on understanding, not memorising
Mental model
Think of a building: HTML is the concrete structure (walls, floors, rooms), CSS is the interior design (colours, layout, fonts), and JavaScript is the electrical system (interactivity, movement, data).
The Answer
HTML defines structure and meaning: what is a heading, what is a paragraph, what is a link.

CSS defines visual presentation: colours, fonts, spacing, layout.

JavaScript defines behaviour: responding to clicks, fetching data, updating the page dynamically.

All three are separate files (or sections) that the browser loads together. A page can technically exist without CSS or JS, but not without HTML.
HTML = structure & meaning.
CSS = visual presentation.
JS = dynamic behaviour.
Mental model
A tag is the punctuation mark (the angle-bracket syntax); an element is the whole unit — opening tag + content + closing tag.
The Answer
A tag is the markup notation: <p> or </p>.

An element is the complete construct: <p>Hello world</p> — opening tag, content, closing tag.

Some elements are void elements (self-closing) — they have no content or closing tag: <img>, <br>, <input>.
<!-- Element with content --> <p>This is a paragraph.</p> <!-- Void element (no closing tag) --> <img src="photo.jpg" alt="A description">
⚠️ "Self-closing" syntax like <img /> is valid in HTML5 but the slash is optional — <img> is correct. In XHTML/JSX it is required.
Tag = the angle-bracket syntax (<p>).
Element = full unit including content.
Void elements have no closing tag.
Mental model
Attributes are settings you pass to an element — they go inside the opening tag and provide extra information the element needs to function or be identified.
The Answer
An attribute is a name/value pair inside the opening tag: name="value".
<a href="https://example.com" target="_blank">Visit site</a> <!-- ^^^^^^^^^^^^^^^^^^^^^^^^^^^^ ^^^^^^^^^^^^^^^^^ --> <!-- href attribute target attribute -->
Common attributes: href (link destination), src (image source), alt (image description), id (unique identifier), class (CSS/JS hook), type (input type).
Attributes go inside the opening tag, never the closing tag.
Values are quoted (double quotes are conventional).
Some attributes are boolean — their presence alone enables the feature: <input disabled>.
Mental model
The DOM is the browser's live in-memory representation of your HTML document as a tree of objects. It's what CSS selectors target and what JavaScript reads and modifies.
The Answer
When a browser loads an HTML file it parses the markup and constructs a Document Object Model — a tree where each element, attribute, and piece of text is a node.

CSS traverses this tree to apply styles. JavaScript manipulates it via document.querySelector(), element.textContent, etc.

The DOM is a live representation: changes made by JS are reflected instantly without reloading the HTML file.
DOM = browser's live tree built from your HTML.
CSS and JS both operate on the DOM, not the raw HTML text.
The DOM can be modified after load — that's how dynamic web apps work.

HTML ist das Gerüst jeder Webseite

HTML (HyperText Markup Language) ist die Standardsprache zur Erstellung von Webseiten. Es beschreibt die Struktur und Bedeutung von Inhalten — nicht deren visuelle Erscheinung. CSS kümmert sich um das Aussehen; JavaScript steuert das Verhalten; HTML definiert, was alles ist.

Jede Webseite, die dein Browser lädt, ist im Kern ein HTML-Dokument — eine einfache Textdatei mit der Endung .html.
HTML verwendet Elemente (Tags wie <p>, <h1>, <a>), um Inhalten Bedeutung zu geben.
Browser parsen HTML in einen Baum namens DOM (Document Object Model) — die lebendige Darstellung, auf die CSS und JS operieren.
HTML ist keine Programmiersprache — es gibt keine Logik, Schleifen oder Variablen. Es ist eine Auszeichnungssprache.
❓
Kernkonzeptfragen
Klicke eine Frage an — auf Verstehen, nicht Auswendiglernen fokussieren
Mentales Modell
Stell dir ein Gebäude vor: HTML ist die Betonstruktur (Wände, Böden, Zimmer), CSS ist die Inneneinrichtung (Farben, Layout, Schriften), und JavaScript ist die Elektroanlage (Interaktivität, Bewegung, Daten).
Die Antwort
HTML definiert Struktur und Bedeutung: was eine Überschrift ist, was ein Absatz ist, was ein Link ist.

CSS definiert die visuelle Darstellung: Farben, Schriften, Abstände, Layout.

JavaScript definiert das Verhalten: auf Klicks reagieren, Daten abrufen, die Seite dynamisch aktualisieren.

Alle drei sind separate Dateien (oder Abschnitte), die der Browser gemeinsam lädt. Eine Seite kann technisch ohne CSS oder JS existieren, aber nicht ohne HTML.
HTML = Struktur & Bedeutung.
CSS = visuelle Darstellung.
JS = dynamisches Verhalten.
Mentales Modell
Ein Tag ist das Satzzeichen (die Spitzklammer-Syntax); ein Element ist die vollständige Einheit — öffnendes Tag + Inhalt + schließendes Tag.
Die Antwort
Ein Tag ist die Markup-Notation: <p> oder </p>.

Ein Element ist das vollständige Konstrukt: <p>Hallo Welt</p> — öffnendes Tag, Inhalt, schließendes Tag.

Manche Elemente sind void-Elemente (selbstschließend) — sie haben keinen Inhalt oder kein schließendes Tag: <img>, <br>, <input>.
Tag = die Spitzklammer-Syntax.
Element = vollständige Einheit inklusive Inhalt.
Void-Elemente haben kein schließendes Tag.
Mentales Modell
Attribute sind Einstellungen, die du einem Element übergibst — sie stehen im öffnenden Tag und liefern zusätzliche Informationen, die das Element zum Funktionieren benötigt.
Die Antwort
Ein Attribut ist ein Name/Wert-Paar im öffnenden Tag: name="wert".

Häufige Attribute: href (Link-Ziel), src (Bildquelle), alt (Bildbeschreibung), id (eindeutiger Bezeichner), class (CSS/JS-Anker), type (Eingabetyp).
Attribute stehen im öffnenden Tag, niemals im schließenden.
Werte werden in Anführungszeichen gesetzt (doppelte Anführungszeichen sind üblich).
Boolesche Attribute aktivieren ein Merkmal durch ihre bloße Anwesenheit: <input disabled>.
