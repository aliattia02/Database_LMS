Lesson 1 · ~30 min

Headings & Paragraphs

Headings create an outline that both users and search engines use to navigate. The correct hierarchy is more important than the visual size.

Screen readers use heading structure to let blind users jump between sections — just like a sighted user would skim a table of contents. Broken heading hierarchy is one of the most common accessibility failures. Search engines also weight <h1> content heavily for SEO.

⚓ Core Insight

Never choose a heading level based on how it looks — choose it based on the document outline. If you want a heading to look smaller, use CSS, not a lower-level tag.

Heading Levels h1–h6
Six levels of headings where <h1> is the most important (page title) and <h6> the least. Each page should have exactly one <h1>. Levels should not be skipped — go from h2 to h3, never h2 to h4.
Paragraph <p>
The standard block-level container for a unit of text. Browsers add default top and bottom margins between paragraphs. Never use <br> tags to create fake paragraph spacing — use real <p> elements.
Line Break <br>
Forces a line break within a block element — for poetry, addresses, or other content where line breaks are part of the meaning. Do not use multiple <br> tags to create vertical spacing — that is a CSS concern.
ElementSemantic RoleTypical Use
<h1>Page titleOne per page — the main topic
<h2>Major sectionMain sections of the page
<h3>Sub-sectionSub-topics within an h2 section
<h4>–<h6>Deeper nestingRarely needed; restructure first
⚠️ Common Pitfall

Using <h3> right after <h1> (skipping <h2>) creates a broken document outline. Screen reader users rely on a continuous heading hierarchy to navigate efficiently.

✅ Scenario

A blog post about French cooking. Structure it with a page title, two sections (Ingredients, Instructions), and a sub-section under Instructions.

HTML
<h1>Classic French Onion Soup</h1> <!-- Page title --> <h2>Ingredients</h2> <!-- Major section --> <p>You will need the following ingredients for 4 servings.</p> <h2>Instructions</h2> <p>Follow these steps carefully for best results.</p> <h3>Preparing the Onions</h3> <!-- Sub-section of Instructions --> <p>Slice the onions thinly and caramelise over low heat.</p>
Lesson 2 · ~30 min

Inline Text Formatting & Lists

HTML provides both semantic formatting (which carries meaning) and presentational formatting (which only affects appearance). Understanding the difference affects accessibility and SEO.

<strong> — Strong Importance
Semantic: marks text as critically important. Screen readers emphasise it with a different tone. Visually renders bold by default, but its meaning is "this matters", not "make this bold".
<em> — Emphasis
Semantic: marks text with stress emphasis — the kind that changes meaning when spoken aloud. Visually renders italic. "I never said she stole the money" vs "I never said she stole the money" — the meaning shifts.
<b> and <i>
Presentational: visually bold/italic with no semantic weight. Use <b> for keywords, product names, or UI strings where bold is a convention, not importance. Use <i> for titles of works, technical terms, or phrases in another language.
<mark>, <small>, <del>, <ins>
<mark> = highlighted/relevant text. <small> = fine print/side comments. <del> = deleted text (strikethrough). <ins> = inserted text (underline). All carry semantic meaning, not just visual style.
<ul> — Unordered List
A list where order does not matter — items are rendered as bullets. Use for: ingredients, features, navigation menus, collections of items without natural sequence.
<ol> — Ordered List
A list where order does matter — items are rendered numbered. Use for: step-by-step instructions, rankings, legal clauses, anything with a sequence. Supports start and reversed attributes.
<li> — List Item
The only valid direct child of <ul> and <ol>. Each item in either list type uses <li>. Lists can be nested: put a full <ul> or <ol> inside an <li> to create sub-lists.
<dl>, <dt>, <dd> — Description List
A specialised list for term–definition pairs: <dl> wraps the list, <dt> is the term, <dd> is the description. Perfect for glossaries, metadata tables, or FAQ-style Q&A.
HTML — List examples
<!-- Unordered list --> <ul> <li>Flour</li> <li>Butter</li> <li>Eggs</li> </ul> <!-- Ordered list --> <ol> <li>Preheat oven to 180°C</li> <li>Mix dry ingredients</li> <li>Add wet ingredients and stir</li> </ol> <!-- Nested list --> <ul> <li>Frontend <ul> <li>HTML</li> <li>CSS</li> </ul> </li> <li>Backend</li> </ul>
⚠️ Common Pitfall

Only <li> elements are valid direct children of <ul> and <ol>. Placing a <p> or <div> directly inside a list (without wrapping it in <li>) is invalid HTML.

Practice Questions
In-depth questions for this topic
Mental model
Ask: "Does the bold serve a meaning, or is it just a visual choice?" If meaning → <strong>. If convention → <b>.
The Answer
<strong> is semantic: it communicates strong importance or urgency. Screen readers may change tone or volume when reading it. Use it for warnings, critical instructions, or key terms where the emphasis changes the meaning.

<b> is presentational: it makes text visually bold without implying any special importance. Use it for product names, keywords, or any case where bold is a stylistic convention rather than a semantic signal.

Both render identically in most browsers. The difference is in meaning for assistive technology and document semantics.
<strong> = semantic importance (affects screen readers).
<b> = visual bold with no extra meaning.
Same rule applies to <em> (semantic stress) vs <i> (visual italic).
Mental model
Ask: "Would re-ordering these items change their meaning?" If yes → <ol>. If no → <ul>.
The Answer
Use <ul> when order doesn't matter — a shopping list, a set of features, a navigation menu.

Use <ol> when the sequence is meaningful — installation steps, cooking instructions, ranking, legal clauses.

The browser renders them differently (bullets vs. numbers), but more importantly the choice communicates intent to screen readers and machines parsing your document.
<ul>: order is irrelevant — bullets.
<ol>: order is meaningful — numbers.
Navigation menus are almost always <ul> — even though the order may feel fixed.

Lektion 1 · ~30 Min

Überschriften & Absätze

Überschriften erzeugen eine Gliederung, die sowohl Nutzer als auch Suchmaschinen zur Navigation verwenden. Die korrekte Hierarchie ist wichtiger als die visuelle Größe.

Überschriftsebenen h1–h6
Sechs Ebenen von Überschriften, wobei <h1> am wichtigsten (Seitentitel) und <h6> am wenigsten wichtig ist. Jede Seite sollte genau eine <h1> haben. Ebenen dürfen nicht übersprungen werden.
Absatz <p>
Der Standard-Block-Container für eine Texteinheit. Verwende niemals <br>-Tags, um falschen Absatzabstand zu erzeugen — verwende echte <p>-Elemente.
⚠️ Häufiger Fehler

<h3> direkt nach <h1> zu verwenden (wobei <h2> übersprungen wird) erzeugt eine fehlerhafte Dokumentgliederung. Screenreader-Nutzer verlassen sich auf eine durchgehende Überschriftenhierarchie.

<ul> — Ungeordnete Liste
Eine Liste, bei der die Reihenfolge keine Rolle spielt — Elemente werden als Aufzählungspunkte gerendert. Verwendung für: Zutaten, Funktionen, Navigationsmenüs.
<ol> — Geordnete Liste
Eine Liste, bei der die Reihenfolge wichtig ist — Elemente werden nummeriert gerendert. Verwendung für: Schritt-für-Schritt-Anleitungen, Rankings, alles mit einer Sequenz.

Übungsfragen
Vertiefende Fragen zu diesem Thema
Mentales Modell
Frage: „Dient die Fettschrift einer Bedeutung oder ist es nur eine visuelle Wahl?" Wenn Bedeutung → <strong>. Wenn Konvention → <b>.
Die Antwort
<strong> ist semantisch: Es vermittelt starke Wichtigkeit oder Dringlichkeit. Screenreader können Ton oder Lautstärke beim Vorlesen ändern.

<b> ist präsentational: Es macht Text visuell fett, ohne besondere Wichtigkeit zu implizieren. Verwende es für Produktnamen, Keywords oder Fälle, bei denen Fettschrift eine stilistische Konvention ist.
<strong> = semantische Wichtigkeit (beeinflusst Screenreader).
<b> = visuelle Fettschrift ohne zusätzliche Bedeutung.
Gleiche Regel gilt für <em> (semantische Betonung) vs <i> (visuelle Kursivschrift).
