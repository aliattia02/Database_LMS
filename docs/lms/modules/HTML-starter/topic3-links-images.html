Lesson 1 · ~25 min

Hyperlinks — Connecting the Web

The anchor element <a> is what makes the web a web. Without links, every page is an island. Understanding the different URL types and link attributes is essential every day of your career.

Links are the most-used HTML element after headings and paragraphs. Getting the URL type wrong (absolute vs relative) breaks navigation across environments. Missing rel="noopener" on external links is a security issue that passes into production daily.

⚓ Core Insight

The href attribute on <a> can hold a URL, a page section anchor, an email address, or a phone number. The element itself is always inline — it flows inside paragraphs and headings.

<a href="..."> — The anchor element
Creates a clickable hyperlink. The href (hypertext reference) attribute contains the destination. The content between the tags becomes the clickable text or element.
Absolute URLs — full path including protocol
Include the full address: https://example.com/about.html. Use for external links — any page on a different domain. Required when linking from email or any context outside your own site.
Relative URLs — path relative to current file
Omit the domain: ./about.html, ../images/logo.png, /contact.html. Use for internal links between pages in the same project. They continue working even after the domain changes.
target="_blank" — open in new tab
Opens the link in a new browser tab or window. Useful for external links so users don't leave your site. Always pair with rel="noopener noreferrer" to prevent security vulnerabilities.
rel="noopener noreferrer" — security attribute
Prevents the opened page from accessing your page via window.opener (a phishing attack vector). noreferrer also hides the referrer URL. Required whenever you use target="_blank".
Special href values
href="#section-id" — jump to an element with that id on the same page. href="mailto:user@example.com" — open the user's email client. href="tel:+49301234567" — dial on mobile devices.
⚠️ Common Mistake

Using target="_blank" without rel="noopener noreferrer" allows the opened page to redirect your original tab to a malicious URL. Always include both attributes together.

✅ Examples — Four link patterns

These four patterns cover 95% of real-world link usage.

HTML
<!-- 1. Internal link (relative URL) --> <a href="./about.html">About us</a> <!-- 2. External link — opens in new tab, with security attributes --> <a href="https://developer.mozilla.org" target="_blank" rel="noopener noreferrer">MDN Web Docs</a> <!-- 3. Same-page anchor — jumps to element with id="contact" --> <a href="#contact">Jump to contact section</a> ... <section id="contact">...</section> <!-- 4. Email and phone links --> <a href="mailto:hello@example.com">Email us</a> <a href="tel:+49301234567">+49 30 1234567</a>

Lesson 2 · ~25 min

Images — Embedding Visual Content

The <img> element is a void element (no closing tag) that embeds images. Its three most important attributes — src, alt, and explicit dimensions — prevent the two most common image bugs: broken layouts and inaccessible content.

src — Image source
Path to the image file — relative or absolute URL. Same rules as link href: use relative paths for images in your project, absolute URLs for images hosted elsewhere.
alt — Alternative text (required)
Describes the image for screen readers, search engines, and broken-image fallback. Write it as a description of what the image communicates, not "image of…". Decorative images: use alt="" (empty string, not omitted) so screen readers skip them.
width and height — Explicit dimensions
Always specify both in HTML (or CSS). The browser reserves space before the image loads, preventing Cumulative Layout Shift (CLS) — a Core Web Vitals metric. Values without a unit are treated as pixels.
loading="lazy" — Deferred loading
Tells the browser to defer loading until the image is near the viewport. Use on all images that are not visible on initial page load. Do not use on hero images or above-the-fold content — it delays the largest contentful paint.
<figure> and <figcaption>
Wrap images (or code, charts, quotes) in <figure> when they have a caption. <figcaption> provides the visible caption. This is the semantic way to associate a caption with its image — screen readers link them correctly.
⚠️ Common Mistake

Omitting alt entirely is different from alt="". Without the attribute at all, screen readers read out the file name (img-2024-03-17.jpg) as a substitute. Always include alt — either descriptive text or an empty string for decorative images.

srcset — Resolution switching
Provide multiple image files at different sizes. The browser picks the most appropriate based on the device screen density and viewport. Use when the image is the same subject at different resolutions (e.g. hero.jpg at 400 w, 800 w, 1200 w).
sizes — Layout width hint
Tells the browser how wide the image will be at different breakpoints, so it can select the right srcset source before CSS loads. Example: sizes="(max-width: 600px) 100vw, 50vw".
<picture> — Art direction
Use when you need to serve a completely different crop or format at different viewport sizes (not just a smaller version of the same image). Contains <source> elements with media conditions, plus a fallback <img>.
ℹ️ Rule of Thumb

Use srcset + sizes for the same image at multiple resolutions. Use <picture> when you need a different crop for mobile vs desktop, or to serve modern formats (WebP, AVIF) with a JPEG fallback.

HTML
<!-- 1. Basic image with all required attributes --> <img src="./images/team-photo.jpg" alt="The Acme team in front of the Berlin office" width="800" height="450" loading="lazy" > <!-- 2. Image with caption using figure/figcaption --> <figure> <img src="./images/chart.png" alt="Bar chart showing Q3 revenue up 23% versus Q2" width="640" height="400" > <figcaption>Q3 2024 revenue increased 23% compared to Q2.</figcaption> </figure> <!-- 3. Responsive image with srcset --> <img src="./images/hero-800.jpg" srcset="./images/hero-400.jpg 400w, ./images/hero-800.jpg 800w, ./images/hero-1200.jpg 1200w" sizes="(max-width: 600px) 100vw, 800px" alt="City skyline at night" width="800" height="500" > <!-- 4. Decorative image — empty alt so screen readers skip it --> <img src="./images/divider-wave.svg" alt="" width="1200" height="40">
Lesson 3 · ~10 min

Audio & Video

HTML5 brought native media elements that work without plugins. The <video> and <audio> elements share a similar attribute model and both support the <source> element for format fallbacks.

controls — Show browser controls
Adds the browser's built-in play/pause/seek/volume controls. Without it, the media is invisible to the user (you'd need to build custom controls with JavaScript). Always include unless building a custom player.
autoplay, muted, loop
Browsers block autoplay with audio unless the video is also muted. Use muted autoplay loop together for background video loops. Never autoplay audio — it is a serious accessibility and usability issue.
<source> — Format fallback
Provide multiple <source> children so browsers that don't support one format try the next. Modern order: WebM first (smaller), then MP4 as fallback. Always include the type attribute so the browser doesn't download files it can't play.
HTML
<!-- Video with fallback sources --> <video controls width="800" height="450" poster="./thumb.jpg"> <source src="./video/intro.webm" type="video/webm"> <source src="./video/intro.mp4" type="video/mp4"> <!-- Fallback text for very old browsers --> Your browser doesn't support HTML5 video. </video> <!-- Audio player --> <audio controls> <source src="./audio/podcast.ogg" type="audio/ogg"> <source src="./audio/podcast.mp3" type="audio/mpeg"> </audio>

Mental model
Think of your project as a house. A relative URL is like saying "go to the room on the left" — it only makes sense from inside the house. An absolute URL is the full postal address — it works from anywhere in the world.
The Answer
An absolute URL contains the full address including the protocol and domain: https://example.com/about.html. It works from any context — another website, an email, a search engine.

A relative URL omits the domain and is resolved relative to the current document's location: ./about.html, ../images/logo.png, /contact.html (root-relative). Use relative URLs for internal links — they survive domain changes and work in local development.
Absolute = full address with protocol — use for external links.
Relative = path only — use for internal links within your project.
/path is root-relative (relative to the domain root, not the current file).
Mental model
The alt attribute has two jobs: describe the image when it can't be seen, and tell screen readers whether the image carries meaning. Empty alt (alt="") is an explicit signal: "this image is decorative, ignore it."
The Answer
alt="" (explicitly empty) tells screen readers to skip the image entirely. This is correct for decorative images — dividers, background textures, icons that have accompanying text labels — where the image adds no information.

Omitting alt entirely is different: screen readers will read out the filename (e.g. "img-2024-03-17.jpg"), which is confusing and unhelpful.

Meaningful images — photos, charts, product images, infographics — need a genuine text description in alt that conveys the same information as the image.
Decorative image: alt="" — screen reader skips it.
Meaningful image: describe what the image communicates, not "image of…".
Never omit the alt attribute — always present, even if empty.
Mental model
When you open a new tab with target="_blank", the new page can reach back and control the original tab through window.opener. It's like handing someone a key to your front door when you thought you were just giving them directions.
The Answer
Without rel="noopener", the page opened in the new tab can access window.opener and redirect your original page to a phishing site — a technique called "reverse tabnapping".

rel="noopener" breaks the reference to window.opener. rel="noreferrer" additionally hides the HTTP Referer header (prevents the destination from knowing where the user came from) and also implies noopener.

Modern browsers now apply noopener by default for cross-origin target="_blank" links, but explicitly including it is still considered best practice.
Without it: the opened page can redirect your original tab.
noopener — prevents window.opener access.
noreferrer — also hides the HTTP Referer header, implies noopener.

Hyperlinks — Das Web vernetzen

Das Anker-Element <a> macht das Web zu einem Web. Ohne Links ist jede Seite eine Insel. Das Verständnis der verschiedenen URL-Typen und Link-Attribute ist täglich im Einsatz.

Links sind das am häufigsten verwendete HTML-Element nach Überschriften und Absätzen. Den falschen URL-Typ zu wählen (absolut vs. relativ) bricht die Navigation in verschiedenen Umgebungen. Fehlendes rel="noopener" bei externen Links ist ein Sicherheitsproblem, das täglich in den Produktionscode gelangt.

⚓ Kernaussage

Das href-Attribut auf <a> kann eine URL, einen Seitenanker, eine E-Mail-Adresse oder eine Telefonnummer enthalten. Das Element selbst ist immer inline — es fließt innerhalb von Absätzen und Überschriften.

<a href="..."> — Das Anker-Element
Erstellt einen klickbaren Hyperlink. Das href-Attribut (Hypertext-Referenz) enthält das Ziel. Der Inhalt zwischen den Tags wird zum klickbaren Text oder Element.
Absolute URLs — vollständige Adresse mit Protokoll
Enthält die vollständige Adresse: https://example.com/about.html. Für externe Links auf anderen Domains verwenden. Notwendig beim Verlinken aus E-Mails oder außerhalb der eigenen Website.
Relative URLs — Pfad relativ zur aktuellen Datei
Domain weglassen: ./about.html, ../images/logo.png, /contact.html. Für interne Links zwischen Seiten im selben Projekt verwenden. Funktionieren auch nach einem Domain-Wechsel.
target="_blank" — In neuem Tab öffnen
Öffnet den Link in einem neuen Browser-Tab. Nützlich für externe Links, damit Nutzer die Seite nicht verlassen. Immer mit rel="noopener noreferrer" kombinieren, um Sicherheitslücken zu vermeiden.
rel="noopener noreferrer" — Sicherheitsattribut
Verhindert, dass die geöffnete Seite über window.opener auf deine Seite zugreift (ein Phishing-Angriffsvektor). noreferrer verbirgt zusätzlich die Referrer-URL. Pflicht bei target="_blank".
Spezielle href-Werte
href="#abschnitt-id" — springt zum Element mit dieser id auf der selben Seite. href="mailto:nutzer@beispiel.de" — öffnet den E-Mail-Client. href="tel:+49301234567" — wählt auf Mobilgeräten.
⚠️ Häufiger Fehler

target="_blank" ohne rel="noopener noreferrer" zu verwenden, erlaubt der geöffneten Seite, deinen ursprünglichen Tab auf eine bösartige URL umzuleiten. Beide Attribute immer gemeinsam verwenden.

HTML
<!-- 1. Interner Link (relative URL) --> <a href="./ueber-uns.html">Über uns</a> <!-- 2. Externer Link — öffnet in neuem Tab, mit Sicherheitsattributen --> <a href="https://developer.mozilla.org" target="_blank" rel="noopener noreferrer">MDN Web Docs</a> <!-- 3. Seitenanker — springt zum Element mit id="kontakt" --> <a href="#kontakt">Zum Kontaktbereich</a> <!-- 4. E-Mail- und Telefon-Links --> <a href="mailto:hallo@beispiel.de">Schreib uns</a> <a href="tel:+49301234567">+49 30 1234567</a>
Lektion 2 · ~25 Min.

Bilder — Visuelle Inhalte einbetten

Das <img>-Element ist ein Void-Element (kein schließendes Tag), das Bilder einbettet. Seine drei wichtigsten Attribute — src, alt und explizite Dimensionen — verhindern die zwei häufigsten Bild-Fehler: kaputte Layouts und unzugängliche Inhalte.

src — Bildquelle
Pfad zur Bilddatei — relative oder absolute URL. Gleiche Regeln wie bei Link-href: relative Pfade für Bilder im eigenen Projekt, absolute URLs für extern gehostete Bilder.
alt — Alternativtext (Pflicht)
Beschreibt das Bild für Screenreader, Suchmaschinen und bei fehlendem Bild. Als Beschreibung formulieren, was das Bild kommuniziert — nicht „Bild von…". Dekorative Bilder: alt="" (leerer String, nicht weglassen), damit Screenreader sie überspringen.
width und height — Explizite Maße
Immer beide angeben (in HTML oder CSS). Der Browser reserviert Platz, bevor das Bild geladen wird — verhindert Cumulative Layout Shift (CLS), eine wichtige Core Web Vitals-Metrik.
loading="lazy" — Verzögertes Laden
Weist den Browser an, das Laden zu verzögern, bis das Bild sich dem Viewport nähert. Für alle Bilder verwenden, die beim ersten Seitenaufruf nicht sichtbar sind. Nicht für Hero-Bilder oder Above-the-fold-Inhalte verwenden.
<figure> und <figcaption>
Bilder (oder Code, Diagramme, Zitate) in <figure> einwickeln, wenn sie eine Beschriftung haben. <figcaption> liefert die sichtbare Bildunterschrift. Screenreader verknüpfen Bild und Beschriftung korrekt.
⚠️ Häufiger Fehler

alt ganz wegzulassen unterscheidet sich von alt="". Ohne das Attribut lesen Screenreader den Dateinamen vor (img-2024-03-17.jpg). Immer alt angeben — entweder mit Beschreibungstext oder leerem String.

srcset — Auflösungswechsel
Stellt mehrere Bilddateien in verschiedenen Größen bereit. Der Browser wählt anhand der Gerätepixeldichte und der Viewport-Breite die passende. Verwendung, wenn dasselbe Motiv in verschiedenen Auflösungen benötigt wird.
sizes — Layout-Breitenhinweis
Gibt dem Browser an, wie breit das Bild bei verschiedenen Breakpoints sein wird, damit er das richtige srcset-Bild vor dem CSS-Laden auswählen kann. Beispiel: sizes="(max-width: 600px) 100vw, 50vw".
<picture> — Art Direction
Verwenden, wenn für verschiedene Viewport-Größen ein komplett anderer Bildausschnitt oder ein anderes Format benötigt wird. Enthält <source>-Elemente mit media-Bedingungen und ein Fallback-<img>.
HTML
<!-- 1. Bild mit allen erforderlichen Attributen --> <img src="./bilder/team-foto.jpg" alt="Das Acme-Team vor dem Berliner Büro" width="800" height="450" loading="lazy" > <!-- 2. Bild mit Bildunterschrift --> <figure> <img src="./bilder/diagramm.png" alt="Balkendiagramm: Q3-Umsatz 23% über Q2" width="640" height="400" > <figcaption>Der Q3-Umsatz 2024 stieg um 23% gegenüber Q2.</figcaption> </figure> <!-- 3. Dekoratives Bild — leeres alt --> <img src="./bilder/trennlinie.svg" alt="" width="1200" height="40">
Lektion 3 · ~10 Min.

Audio & Video

HTML5 brachte native Medienelemente, die ohne Plugins funktionieren. <video> und <audio> teilen ein ähnliches Attributmodell und unterstützen beide das <source>-Element für Format-Fallbacks.

controls — Browser-Steuerelemente anzeigen
Fügt die eingebauten Browser-Steuerelemente (Abspielen/Pause/Seek/Lautstärke) hinzu. Ohne diese Attribute ist das Medium für den Nutzer unsichtbar. Immer verwenden, wenn kein eigener Player per JavaScript gebaut wird.
autoplay, muted, loop
Browser blockieren autoplay mit Ton, außer wenn das Video auch muted ist. muted autoplay loop zusammen für Hintergrundvideo-Schleifen verwenden. Niemals Audio automatisch abspielen — das ist eine schwere Usability- und Barrierefreiheitsverletzung.
<source> — Format-Fallback
Mehrere <source>-Kinder angeben, damit Browser, die ein Format nicht unterstützen, das nächste versuchen. Moderne Reihenfolge: WebM zuerst (kleiner), dann MP4 als Fallback. Immer type-Attribut angeben.

Mentales Modell
Stell dir dein Projekt als Haus vor. Eine relative URL ist wie „geh in den Raum links" — funktioniert nur von innen. Eine absolute URL ist die vollständige Postadresse — funktioniert von überall.
Die Antwort
Eine absolute URL enthält die vollständige Adresse mit Protokoll und Domain: https://beispiel.de/ueber-uns.html. Funktioniert in jedem Kontext — andere Website, E-Mail, Suchmaschine.

Eine relative URL lässt die Domain weg und wird relativ zum Speicherort des aktuellen Dokuments aufgelöst: ./ueber-uns.html, ../bilder/logo.png, /kontakt.html. Für interne Links verwenden — sie funktionieren auch nach einem Domain-Wechsel und in der lokalen Entwicklung.
Absolut = vollständige Adresse mit Protokoll — für externe Links.
Relativ = nur Pfad — für interne Links im eigenen Projekt.
/pfad ist root-relativ (relativ zur Domain-Root, nicht zur aktuellen Datei).
Mentales Modell
Das alt-Attribut hat zwei Aufgaben: das Bild beschreiben, wenn es nicht gesehen werden kann, und Screenreadern signalisieren, ob das Bild Bedeutung trägt. Leeres alt (alt="") bedeutet explizit: „Dieses Bild ist dekorativ, ignoriere es."
Die Antwort
alt="" (explizit leer) weist Screenreader an, das Bild zu überspringen. Korrekt für dekorative Bilder — Trennlinien, Hintergrundtexturen, Icons mit Begleittext.

alt ganz wegzulassen ist anders: Screenreader lesen dann den Dateinamen vor (z.B. „img-2024-03-17.jpg"), was verwirrend ist.

Bedeutungsvolle Bilder — Fotos, Diagramme, Produktbilder, Infografiken — benötigen eine echte Textbeschreibung in alt.
Dekoratives Bild: alt="" — Screenreader überspringt es.
Bedeutungsvolles Bild: beschreiben, was es kommuniziert — nicht „Bild von…".
alt niemals weglassen — immer angeben, auch wenn leer.
Mentales Modell
Wenn du mit target="_blank" einen neuen Tab öffnest, kann die neue Seite über window.opener auf deinen ursprünglichen Tab zugreifen. Es ist wie jemandem einen Schlüssel zu deiner Haustür zu geben, während du dachtest, du gibst ihm nur eine Wegbeschreibung.
Die Antwort
Ohne rel="noopener" kann die im neuen Tab geöffnete Seite über window.opener deinen ursprünglichen Tab auf eine Phishing-Seite umleiten — „Reverse Tabnapping".

rel="noopener" unterbricht die Referenz zu window.opener. rel="noreferrer" verbirgt zusätzlich den HTTP-Referer-Header und impliziert noopener.

Moderne Browser wenden noopener standardmäßig an, aber das explizite Angeben gilt weiterhin als Best Practice.
Ohne es: die geöffnete Seite kann deinen ursprünglichen Tab umleiten.
noopener — verhindert window.opener-Zugriff.
noreferrer — verbirgt auch den HTTP-Referer-Header, impliziert noopener.
