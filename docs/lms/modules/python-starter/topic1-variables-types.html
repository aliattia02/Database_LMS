Assignment & Naming
How Python binds names to objects — and the rules that govern them

How variable assignment works

In Python, a variable is not a container — it is a name tag attached to an object in memory. When you write x = 42, Python creates the integer object 42 and makes the name x point to it. Multiple names can point to the same object.

x = 42 # x points to int object 42 y = x # y points to the same object x = 100 # x now points to a NEW object; y still points to 42 print(y) # → 42
Naming rules: names must start with a letter or underscore, contain only letters, digits, and underscores, and must not clash with a keyword.
Use snake_case for variable names by convention — user_age, not UserAge or userAge.
Python is case-sensitive: Score, score, and SCORE are three different names.

The Six Scalar Types
Know them by name, literal syntax, and typical use

Python's built-in scalar types at a glance

Python has six fundamental scalar (single-value) types. Every value you work with belongs to one of them.

Type Keyword Example literal Use when…
int int 42, -7, 0 Whole numbers, counts, indices
float float 3.14, -0.5, 1e6 Decimal / scientific values
str str "hello", 'world' Text, identifiers, messages
bool bool True, False Flags, conditions, toggles
NoneType None None Absence of a value, optional returns
complex complex 2+3j Scientific / mathematical computation

Use type(value) to inspect the type of any object at runtime.

print(type(42)) # <class 'int'> print(type(3.14)) # <class 'float'> print(type("hello")) # <class 'str'> print(type(True)) # <class 'bool'> print(type(None)) # <class 'NoneType'>
bool is a subclass of int — True == 1 and False == 0 evaluate to True, which can cause surprises in arithmetic.
None is not zero, not an empty string, and not False — it is its own singleton type.
Python integers have arbitrary precision — they never overflow, unlike integers in C or Java.

Type Conversion
Safe and unsafe ways to convert between types

Explicit vs implicit conversion

Implicit conversion (coercion) happens automatically when Python can promote a type safely — for example, adding an int to a float yields a float. Explicit conversion uses constructor functions like int(), float(), str(), and bool().

# Implicit — Python widens int to float automatically result = 3 + 1.5 # → 4.5 (float) # Explicit — you control the conversion age_str = "28" age_int = int(age_str) # → 28 (int) age_float = float(age_str) # → 28.0 (float) back_to_str = str(age_int) # → "28" (str) # Danger — fails at runtime int("hello") # ValueError: invalid literal
Always wrap int() or float() conversions from user input in a try/except ValueError block — strings like "abc" raise an error.
int(3.9) truncates toward zero — it does not round. Use round() if rounding is what you need.
Truthiness: bool(0), bool(""), bool(None), bool([]) all return False; most other values return True.

String Basics & f-strings
Working with text — the type you'll use most

String literals and f-string formatting

Strings are sequences of Unicode characters. Use single or double quotes interchangeably; use triple quotes for multi-line strings. The modern way to embed values into strings is an f-string (formatted string literal).

name = "Alice" score = 97.5 # f-string — prefix with f, embed expressions in {} message = f"Hello, {name}! Your score is {score:.1f}." print(message) # → Hello, Alice! Your score is 97.5. # Useful format specifiers f"{3.14159:.2f}" # → "3.14" (2 decimal places) f"{1000000:,}" # → "1,000,000" (thousands separator) f"{'left':<10}" # → "left " (left-align in 10 chars)
f-strings are evaluated at runtime, so you can put any valid Python expression inside the braces: f"{2 ** 10}" → "1024".
Strings are immutable — you can never change a character in place; string methods always return a new string.
Prefer f-strings over % formatting or .format() — they are faster and more readable.

Practice Questions
Typical interview and self-check questions on variables & types
Mental model
Interviewers ask this to see whether you distinguish Python's types from lower-level languages. Mention that Python types are classes, and that every value is an object.
The Answer
The six scalar types are int, float, str, bool, NoneType (value: None), and complex. You check a value's type with type(value), which returns the class object (e.g. <class 'int'>). To check membership use isinstance(value, int) — it respects inheritance, so it returns True for booleans when the target type is int.
⚠️ Don't confuse type(x) == int with isinstance(x, int). The former fails for subclasses; the latter is usually the right check in production code.
type() is for inspection; isinstance() is for conditional logic.
bool is a subclass of int, so isinstance(True, int) returns True.
Mental model
Think of it as the difference between a labelled box (C/Java) and a name tag on an object (Python). The object exists independently; the name just points to it.
The Answer
In C/Java a variable is a fixed memory slot with a declared type — changing the value overwrites that slot. In Python, a variable is a name binding: the name points to an object on the heap, and that object carries its own type. Reassigning a name just makes it point to a different object; the old object remains until garbage-collected. Python variables have no declared type — the object does.
Python is dynamically typed — type is attached to the object, not the variable name.
Multiple names can reference the same object; use id(x) to inspect the object's memory address.
Mental model
Python uses "truthiness" pervasively in conditionals. Knowing the falsy set lets you write idiomatic if value: guards instead of verbose equality checks.
The Answer
The falsy values in Python are: False, 0, 0.0, 0j, None, "" (empty string), [] (empty list), {} (empty dict), () (empty tuple), and set(). Everything else is truthy. This matters because Python's if, while, and boolean operators use truthiness — if user_name: is idiomatic for checking a non-empty string.
⚠️ if value == False: is usually wrong — it passes for 0 and 0.0 too, since False == 0. Use if value is False: when you specifically need the boolean singleton.
Memorize: None, 0, "", empty containers — all falsy.
Prefer if items: over if len(items) > 0: — it is more Pythonic.
Mental model
User input is always a string. Converting it is one of the first things every real script does — and the most common source of beginner crashes.
The Answer
Wrap the conversion in a try/except ValueError block. int() raises ValueError for any string that is not a valid integer literal, and TypeError if you pass None. Handle both:
raw = input("Enter a number: ") try: value = int(raw) except (ValueError, TypeError): print(f"'{raw}' is not a valid integer.") value = 0 # sensible default
⚠️ int("3.7") raises ValueError — int() does not accept decimal strings. Convert to float first, then to int if needed: int(float("3.7")).
Always validate user input before conversion — never assume it is a number.
int(float("3.7")) → 3 (truncates, does not round).
Mental model
None means "no value" — it is the explicit absence of a result, not a zero or an empty placeholder.
The Answer
None is the sole instance of NoneType. It represents the intentional absence of a value — for example, a function that doesn't return anything implicitly returns None. Unlike 0 (a valid count), False (a boolean result), or "" (an empty but valid string), None signals "not yet set" or "not applicable". Always test for it with is None, not == None.
Use if x is None: — not if x == None: — because == can be overridden by custom classes.
Functions that don't hit a return statement implicitly return None.
Mental model
f-strings are evaluated at the point they are created. Think of the braces as mini-Python expressions embedded directly in the string.
The Answer
An f-string is prefixed with f and evaluates expressions inside {} at runtime. They are faster than .format(), easier to read, and support full Python expressions including function calls and format specifiers: f"{price:.2f}", f"{name.upper()}". They are available from Python 3.6 onward and are now the preferred formatting approach.
Any valid Python expression works inside the braces: f"{2**8}" → "256".
Format spec after colon: :.2f for 2 decimal places, :, for thousands separator, :>10 for right-align.

Zuweisung & Benennung
Wie Python Namen an Objekte bindet — und die Regeln dafür

Wie Variablenzuweisung funktioniert

In Python ist eine Variable kein Behälter — sie ist ein Namensschild, das an ein Objekt im Speicher geheftet wird. Wenn du x = 42 schreibst, erstellt Python das Integer-Objekt 42 und lässt den Namen x darauf zeigen. Mehrere Namen können auf dasselbe Objekt zeigen.

x = 42 # x zeigt auf das int-Objekt 42 y = x # y zeigt auf dasselbe Objekt x = 100 # x zeigt jetzt auf ein NEUES Objekt; y zeigt noch auf 42 print(y) # → 42
Namensregeln: Namen müssen mit einem Buchstaben oder Unterstrich beginnen, dürfen nur Buchstaben, Ziffern und Unterstriche enthalten und dürfen kein Schlüsselwort sein.
Verwende snake_case für Variablennamen — benutzer_alter, nicht BenutzerAlter.
Python unterscheidet Groß- und Kleinschreibung: Score, score und SCORE sind drei verschiedene Namen.

Die sechs skalaren Typen
Nach Name, Literal-Syntax und typischem Einsatz kennen

Pythons eingebaute Skalartypen im Überblick

Python hat sechs fundamentale Skalartypen (Einzelwert-Typen). Jeder Wert, mit dem du arbeitest, gehört zu einem davon.

Typ Schlüsselwort Beispiel-Literal Verwende wenn…
int int 42, -7, 0 Ganze Zahlen, Zählwerte, Indizes
float float 3.14, -0.5, 1e6 Dezimal-/wissenschaftliche Werte
str str "hallo", 'welt' Text, Bezeichner, Nachrichten

Verwende type(wert), um den Typ eines Objekts zur Laufzeit zu prüfen.

print(type(42)) # <class 'int'> print(type(3.14)) # <class 'float'> print(type("hallo")) # <class 'str'> print(type(True)) # <class 'bool'> print(type(None)) # <class 'NoneType'>
bool ist eine Unterklasse von int — True == 1 und False == 0 ergeben True, was bei Arithmetik überraschend sein kann.
None ist weder null noch ein leerer String noch False — es ist ein eigener Singleton-Typ.
Python-Integers haben beliebige Präzision — sie laufen nie über, anders als in C oder Java.

Typkonvertierung
Sichere und unsichere Wege zur Umwandlung zwischen Typen

Explizite vs. implizite Konvertierung

Implizite Konvertierung erfolgt automatisch, wenn Python einen Typ sicher erweitern kann — z. B. beim Addieren von int und float entsteht ein float. Explizite Konvertierung verwendet Konstruktorfunktionen wie int(), float(), str() und bool().

# Implizit — Python erweitert int zu float automatisch ergebnis = 3 + 1.5 # → 4.5 (float) # Explizit — du steuerst die Konvertierung alter_str = "28" alter_int = int(alter_str) # → 28 (int) alter_float = float(alter_str) # → 28.0 (float) zurueck_str = str(alter_int) # → "28" (str) # Gefahr — schlägt zur Laufzeit fehl int("hallo") # ValueError: invalid literal
Wrape int()- oder float()-Konvertierungen von Benutzereingaben immer in einem try/except ValueError-Block.
int(3.9) schneidet ab in Richtung null — es rundet nicht. Verwende round() wenn Runden gewünscht ist.
Wahrheitswerte: bool(0), bool(""), bool(None), bool([]) geben alle False zurück.

String-Grundlagen & f-Strings
Mit Text arbeiten — der Typ, den du am häufigsten verwendest

String-Literale und f-String-Formatierung

Strings sind Folgen von Unicode-Zeichen. Einfache und doppelte Anführungszeichen sind austauschbar; dreifache Anführungszeichen ermöglichen mehrzeilige Strings. Die moderne Methode zum Einbetten von Werten in Strings ist ein f-String.

name = "Alice" punkte = 97.5 # f-String — Präfix f, Ausdrücke in {} nachricht = f"Hallo, {name}! Deine Punktzahl ist {punkte:.1f}." print(nachricht) # → Hallo, Alice! Deine Punktzahl ist 97.5. # Nützliche Format-Spezifizierer f"{3.14159:.2f}" # → "3.14" (2 Nachkommastellen) f"{1000000:,}" # → "1,000,000" (Tausendertrennzeichen) f"{'links':<10}" # → "links " (linksbündig, 10 Zeichen)
f-Strings werden zur Laufzeit ausgewertet — du kannst beliebige Python-Ausdrücke in den Klammern verwenden.
Strings sind unveränderlich — String-Methoden geben immer einen neuen String zurück.
Bevorzuge f-Strings gegenüber %-Formatierung oder .format() — sie sind schneller und lesbarer.

Übungsfragen
Typische Interview- und Selbstprüfungsfragen zu Variablen & Typen
Mentales Modell
Interviewer stellen diese Frage, um zu sehen, ob du Pythons Typen von Low-Level-Sprachen unterscheidest. Erwähne, dass Python-Typen Klassen sind und jeder Wert ein Objekt ist.
Die Antwort
Die sechs Skalartypen sind int, float, str, bool, NoneType (Wert: None) und complex. Den Typ eines Werts prüfst du mit type(wert). Für Bedingungen verwende isinstance(wert, int) — es berücksichtigt Vererbung, sodass es für Booleans True zurückgibt, wenn der Zieltyp int ist.
⚠️ Verwechsle nicht type(x) == int mit isinstance(x, int). Ersteres schlägt bei Unterklassen fehl; letzteres ist die richtige Wahl im Produktionscode.
type() dient der Inspektion; isinstance() für bedingte Logik.
bool ist eine Unterklasse von int, daher gibt isinstance(True, int) True zurück.
Mentales Modell
Stell dir den Unterschied zwischen einer beschrifteten Box (C/Java) und einem Namensschild an einem Objekt (Python) vor.
Die Antwort
In C/Java ist eine Variable ein fixer Speicherplatz mit einem deklarierten Typ. In Python ist eine Variable eine Namensbindung: der Name zeigt auf ein Objekt im Heap, und das Objekt trägt seinen eigenen Typ. Python-Variablen haben keinen deklarierten Typ — das Objekt hat ihn.
Python ist dynamisch typisiert — der Typ gehört dem Objekt, nicht dem Variablennamen.
Mehrere Namen können auf dasselbe Objekt zeigen; id(x) zeigt die Speicheradresse.
Mentales Modell
Python verwendet „Truthiness" überall in Bedingungen. Die falsy-Menge zu kennen ermöglicht idiomatische if wert:-Prüfungen.
Die Antwort
Die falsy-Werte in Python sind: False, 0, 0.0, 0j, None, "" (leerer String), [] (leere Liste), {} (leeres Dict), () (leeres Tupel) und set(). Alles andere ist truthy. Das ist wichtig, weil Pythons if, while und boolesche Operatoren Truthiness verwenden — if benutzername: ist idiomatisch für die Prüfung auf einen nicht-leeren String.
⚠️ if wert == False: ist meist falsch — es trifft auch für 0 und 0.0 zu, da False == 0. Verwende if wert is False: wenn du den booleschen Singleton benötigst.
Merke: None, 0, "", leere Container — alle falsy.
Bevorzuge if elemente: gegenüber if len(elemente) > 0:.
Mentales Modell
Benutzereingaben sind immer Strings. Sie zu konvertieren ist eine der ersten Aufgaben jedes echten Skripts — und die häufigste Quelle von Anfängerfehlern.
Die Antwort
Wrape die Konvertierung in einem try/except ValueError-Block. int() wirft ValueError für jeden String, der kein gültiges Integer-Literal ist, und TypeError wenn du None übergibst:
eingabe = input("Zahl eingeben: ") try: wert = int(eingabe) except (ValueError, TypeError): print(f"'{eingabe}' ist keine gültige Ganzzahl.") wert = 0 # sinnvoller Standardwert
⚠️ int("3.7") wirft ValueError — int() akzeptiert keine Dezimalstrings. Konvertiere zuerst zu float: int(float("3.7")).
Validiere Benutzereingaben immer vor der Konvertierung.
int(float("3.7")) → 3 (abschneiden, nicht runden).
Mentales Modell
None bedeutet „kein Wert" — es ist die explizite Abwesenheit eines Ergebnisses, kein null oder leerer Platzhalter.
Die Antwort
None ist die einzige Instanz von NoneType. Es steht für die bewusste Abwesenheit eines Werts — eine Funktion ohne return gibt implizit None zurück. Im Gegensatz zu 0 (gültige Zahl), False (boolesches Ergebnis) oder "" (leerer, aber gültiger String) signalisiert None „noch nicht gesetzt" oder „nicht zutreffend". Teste immer mit is None, nicht mit == None.
Verwende if x is None: — nicht if x == None:.
Funktionen ohne return geben implizit None zurück.
Mentales Modell
f-Strings werden bei ihrer Erstellung ausgewertet. Die geschweiften Klammern sind Mini-Python-Ausdrücke, direkt in den String eingebettet.
Die Antwort
Ein f-String trägt das Präfix f und wertet Ausdrücke in {} zur Laufzeit aus. Sie sind schneller als .format(), lesbarer und unterstützen vollständige Python-Ausdrücke inklusive Funktionsaufrufe und Formatspezifizierer: f"{preis:.2f}", f"{name.upper()}". Sie stehen ab Python 3.6 zur Verfügung und sind die bevorzugte Formatierungsmethode.
Beliebige Python-Ausdrücke funktionieren in den Klammern: f"{2**8}" → "256".
Formatspezifizierer nach dem Doppelpunkt: :.2f für 2 Nachkommastellen, :, für Tausendertrennzeichen.

الإسناد والتسمية
كيف يربط Python الأسماء بالكائنات — والقواعد التي تحكم ذلك

كيف تعمل إسناد المتغيرات

في Python، المتغير ليس وعاءً — بل هو بطاقة اسم تُلصق على كائن في الذاكرة. عندما تكتب x = 42، يُنشئ Python كائن العدد الصحيح 42 ويجعل الاسم x يشير إليه. يمكن لأسماء متعددة الإشارة إلى نفس الكائن.

x = 42 # x يشير إلى كائن int رقم 42 y = x # y يشير إلى نفس الكائن x = 100 # x يشير الآن إلى كائن جديد؛ y لا يزال يشير إلى 42 print(y) # → 42
قواعد التسمية: يجب أن تبدأ الأسماء بحرف أو شرطة سفلية، وتحتوي فقط على حروف وأرقام وشرطات سفلية، ولا تتعارض مع كلمة محجوزة.
استخدم snake_case بالاتفاقية — user_age وليس UserAge.
Python حساس لحالة الأحرف: Score وscore وSCORE ثلاثة أسماء مختلفة.

تحويل الأنواع
الطرق الآمنة وغير الآمنة للتحويل بين الأنواع

التحويل الصريح مقابل الضمني

يحدث التحويل الضمني تلقائيًا عندما يستطيع Python ترقية النوع بأمان — مثل إضافة int إلى float ينتج float. يستخدم التحويل الصريح دوال البناء مثل int() وfloat() وstr() وbool().

# ضمني — يُرقّي Python تلقائيًا int إلى float result = 3 + 1.5 # → 4.5 (float) # صريح — أنت تتحكم في التحويل age_str = "28" age_int = int(age_str) # → 28 (int) age_float = float(age_str) # → 28.0 (float) back_to_str = str(age_int) # → "28" (str) # خطر — يفشل في وقت التشغيل int("hello") # ValueError: invalid literal
دائمًا ضع تحويلات int() أو float() من إدخال المستخدم داخل كتلة try/except ValueError.
int(3.9) يُقتطع نحو الصفر — لا يُقرّب. استخدم round() إذا كنت تريد التقريب.
القيم الكاذبة: bool(0) وbool("") وbool(None) وbool([]) تُعيد كلها False.

أساسيات النصوص وf-strings
العمل مع النص — النوع الذي ستستخدمه أكثر من غيره

حرفيات النصوص وتنسيق f-string

النصوص هي تسلسلات من أحرف Unicode. استخدم علامات الاقتباس المفردة أو المزدوجة بالتبادل؛ واستخدم علامات الاقتباس الثلاثية للنصوص متعددة الأسطر. الطريقة الحديثة لتضمين القيم في النصوص هي f-string.

name = "Alice" score = 97.5 # f-string — البادئة f، التعبيرات في {} message = f"Hello, {name}! Your score is {score:.1f}." print(message) # → Hello, Alice! Your score is 97.5. # محددات تنسيق مفيدة f"{3.14159:.2f}" # → "3.14" (منزلتان عشريتان) f"{1000000:,}" # → "1,000,000" (فاصل الآلاف) f"{'left':<10}" # → "left " (محاذاة يسار 10 أحرف)
يتم تقييم f-strings في وقت التشغيل — يمكنك وضع أي تعبير Python صالح بين الأقواس.
النصوص غير قابلة للتغيير — دوال النصوص تُعيد دائمًا نصًا جديدًا.
فضّل f-strings على تنسيق % أو .format() — إنها أسرع وأوضح.

أسئلة التدريب
أسئلة مقابلات واختبارات ذاتية نموذجية حول المتغيرات والأنواع
النموذج الذهني
يطرح المحاورون هذا السؤال ليروا ما إذا كنت تميز أنواع Python عن اللغات ذات المستوى المنخفض. اذكر أن أنواع Python هي فئات وأن كل قيمة هي كائن.
الإجابة
الأنواع الستة البسيطة هي int وfloat وstr وbool وNoneType (القيمة: None) وcomplex. تتحقق من نوع القيمة باستخدام type(value). للشروط استخدم isinstance(value, int) — فهو يحترم الوراثة، لذلك يُعيد True للقيم البوليانية عندما النوع المستهدف هو int.
⚠️ لا تخلط بين type(x) == int وisinstance(x, int). الأول يفشل مع الفئات الفرعية؛ الثاني هو الاختيار الصحيح في كود الإنتاج.
type() للفحص؛ isinstance() للمنطق الشرطي.
bool فئة فرعية من int، لذلك isinstance(True, int) تُعيد True.
النموذج الذهني
فكر في الفرق بين صندوق موسوم (C/Java) وبطاقة اسم على كائن (Python).
الإجابة
في C/Java، المتغير هو فتحة ذاكرة ثابتة مع نوع معلَن. في Python، المتغير هو ارتباط اسم: الاسم يشير إلى كائن في الكومة، والكائن يحمل نوعه الخاص. إعادة تعيين الاسم تجعله يشير فقط إلى كائن مختلف. متغيرات Python ليس لها نوع معلَن — الكائن له.
Python ذو كتابة ديناميكية — النوع مرتبط بالكائن لا باسم المتغير.
أسماء متعددة يمكنها الإشارة إلى نفس الكائن؛ id(x) يُظهر عنوان الذاكرة.
النموذج الذهني
يستخدم Python "القيمة الحقيقية" (truthiness) بكثرة في الشروط. معرفة مجموعة القيم الكاذبة يتيح لك كتابة if value: بشكل اصطلاحي.
الإجابة
القيم الكاذبة في Python هي: False و0 و0.0 و0j وNone و"" (نص فارغ) و[] (قائمة فارغة) و{} (قاموس فارغ) و() (tuple فارغ) وset(). كل شيء آخر هو صادق. هذا مهم لأن if وwhile والمشغّلات البوليانية في Python تستخدم القيمة الحقيقية.
⚠️ if value == False: عادةً خاطئ — إنه يمر أيضًا لـ0 و0.0. استخدم if value is False: عند الحاجة للـsingleton البوليان تحديدًا.
احفظ: None و0 و"" والحاويات الفارغة — كلها كاذبة.
فضّل if items: على if len(items) > 0:.
النموذج الذهني
إدخال المستخدم دائمًا نص. تحويله هو أحد أول ما تفعله كل سكريبت حقيقية — وأكثر أسباب أخطاء المبتدئين شيوعًا.
الإجابة
ضع التحويل داخل كتلة try/except ValueError. تُطلق int() خطأ ValueError لأي نص ليس حرفيًا صحيحًا صالحًا، وTypeError عند تمرير None:
raw = input("Enter a number: ") try: value = int(raw) except (ValueError, TypeError): print(f"'{raw}' is not a valid integer.") value = 0 # قيمة افتراضية منطقية
⚠️ int("3.7") تُطلق ValueError — لا تقبل int() النصوص العشرية. حوّل أولًا إلى float: int(float("3.7")).
تحقق دائمًا من إدخال المستخدم قبل التحويل.
int(float("3.7")) → 3 (اقتطاع لا تقريب).
النموذج الذهني
None يعني "لا قيمة" — إنه الغياب الصريح للنتيجة، وليس صفرًا أو نصًا فارغًا.
الإجابة
None هو النسخة الوحيدة من NoneType. يمثّل الغياب المقصود للقيمة — على سبيل المثال، الدالة التي لا تُعيد شيئًا تُعيد ضمنيًا None. على خلاف 0 (عدد صحيح صالح) أو False (نتيجة بوليانية) أو "" (نص فارغ لكن صالح)، يُشير None إلى "لم يُحدَّد بعد" أو "غير قابل للتطبيق". اختبر دائمًا باستخدام is None لا == None.
استخدم if x is None: — لا if x == None:.
الدوال التي لا تصل إلى جملة return تُعيد ضمنيًا None.
النموذج الذهني
تُقيَّم f-strings عند إنشائها. فكّر في الأقواس المعقوفة كتعبيرات Python مصغّرة مضمّنة مباشرة في النص.
الإجابة
f-string تبدأ بالبادئة f وتُقيّم التعبيرات داخل {} في وقت التشغيل. إنها أسرع من .format() وأوضح، وتدعم تعبيرات Python الكاملة بما فيها استدعاءات الدوال ومحددات التنسيق: f"{price:.2f}"، f"{name.upper()}". متاحة منذ Python 3.6 وهي الطريقة المفضّلة للتنسيق.
أي تعبير Python صالح يعمل داخل الأقواس: f"{2**8}" → "256".
محدد التنسيق بعد النقطتين: :.2f لمنزلتين عشريتين، :, لفاصل الآلاف.
