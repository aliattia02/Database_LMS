Control Flow

Master conditional branching with if/elif/else, iterate with for and while loops, and use break and continue to steer execution — the logic engine behind every Python program.

🔀
if / elif / else
Conditional branching — making decisions in code

Branching with if, elif, and else

Python evaluates the first condition that is truthy and runs that block. All other branches are skipped. Indentation (4 spaces) is mandatory — it replaces the curly braces used in other languages.

score = 78 if score >= 90: grade = "A" elif score >= 75: grade = "B" # ← this branch runs; score is 78 elif score >= 60: grade = "C" else: grade = "F" # fallback if nothing above matched print(grade) # → B
Comparison operators: ==, !=, <, <=, >, >=. Use == for equality, never = (that's assignment).
Logical operators: and, or, not. Python uses words, not && / ||.
You can chain comparisons naturally: 0 <= x < 100 is valid Python and reads exactly as in mathematics.
elif and else are optional. A bare if with no else is perfectly valid.

for Loops
Iterating over sequences — the most common loop in Python

Iterating with for and range()

A for loop in Python iterates over any iterable — a list, string, range, tuple, dictionary, or file — one item at a time. You never need to manage an index counter manually.

# Loop over a list fruits = ["apple", "banana", "cherry"] for fruit in fruits: print(fruit) # apple, then banana, then cherry # Loop a fixed number of times with range() for i in range(5): # 0, 1, 2, 3, 4 print(i) # range(start, stop, step) for n in range(2, 11, 2): # 2, 4, 6, 8, 10 print(n) # Loop with index AND value — use enumerate() for idx, val in enumerate(fruits): print(idx, val) # 0 apple, 1 banana, 2 cherry
range(stop) → 0 to stop−1. range(start, stop) → start to stop−1. range(start, stop, step) → every step-th value.
range is lazy — it does not build a list in memory. Use list(range(5)) if you need an actual list.
Prefer enumerate() over manually tracking an index variable — it is more readable and less error-prone.
To loop over two lists in parallel, use zip(list_a, list_b).
🔄
while Loops
Repeating until a condition becomes false

Repeating with while

A while loop runs its body as long as its condition is truthy. Use it when you do not know ahead of time how many iterations are needed — for example, reading user input until it is valid, or polling until a resource is ready.

# Classic countdown count = 5 while count > 0: print(count) count -= 1 print("Done!") # → 5 4 3 2 1 Done! # Validate user input while True: # infinite loop — exited with break raw = input("Enter a positive number: ") if raw.isdigit() and int(raw) > 0: number = int(raw) break print("Invalid — try again.")
Always make sure the loop condition will eventually become False, or use break — an infinite loop freezes your program.
The while True: … break pattern is idiomatic Python for "keep asking until valid input is received".
If you find yourself writing a while loop with a counter variable, a for … range() is usually clearer.

break, continue & Loop else
Fine-grained control inside loops

Steering loops with break, continue, and else

break exits the loop immediately. continue skips the rest of the current iteration and jumps to the next one. The optional else clause on a loop runs only if the loop was not terminated by break.

# break — stop as soon as the target is found numbers = [3, 7, 2, 9, 4] for n in numbers: if n == 9: print("Found 9!") break # exits loop; else clause does NOT run # continue — skip even numbers, print only odds for n in range(1, 10): if n % 2 == 0: continue # jump to next iteration print(n) # → 1 3 5 7 9 # loop else — runs only if break was never hit for n in numbers: if n == 99: print("Found 99!") break else: print("99 not found in list") # ← this runs
break exits the innermost loop only. To break out of nested loops, use a flag variable or refactor into a function and use return.
continue is useful for filtering: skip items you don't care about and keep the happy-path logic un-nested.
The loop else is Python-specific and often surprises developers from other languages. Use it for "search and report if not found" patterns.

Quick reference

Keyword Where it can appear What it does
break for or while Exits the innermost loop immediately
continue for or while Skips to the next iteration
else (loop) After for or while Runs only if the loop finished without hitting break
pass Anywhere a statement is required Does nothing — placeholder for an empty block

Practice Questions
Typical interview and self-check questions on control flow
Mental model
Python doesn't require a boolean — any value can stand in as a condition. Knowing the falsy set lets you write cleaner, more idiomatic guards.
The Answer
Python converts the condition to a boolean using the object's __bool__ method. The falsy values are: False, 0, 0.0, 0j, None, "", [], {}, (), and set(). Every other value is truthy. This means if my_list: is the idiomatic way to check whether a list is non-empty — no need for len(my_list) > 0.
⚠️ if value == False: also matches 0 and 0.0. Use if value is False: when you specifically need the boolean singleton.
Memorise the falsy set: None, 0, empty string, empty container.
Prefer if items: over if len(items) != 0:.
Mental model
A for loop says "do this for each item in a sequence". A while loop says "keep doing this until a condition changes". The key question is: do you know the collection up front?
The Answer
Use a for loop when you have a known iterable (list, range, string, file) — it is cleaner and removes the risk of off-by-one errors. Use a while loop when the number of iterations depends on a condition evaluated at runtime, such as waiting for valid user input, polling a sensor, or reading a stream until EOF. In practice, for loops are far more common in Python.
for loop → known collection; while loop → unknown duration.
A while loop with a counter (i = 0; while i < 10: i += 1) should almost always be a for i in range(10): instead.
Mental model
Think of range as a lazy arithmetic sequence — it remembers the start, stop, and step, and computes each value on demand. It never holds all the numbers in memory at once.
The Answer
range() returns a range object (a lazy sequence), not a list. Its three forms are: range(stop) → 0 up to stop−1; range(start, stop) → start up to stop−1; range(start, stop, step) → every step-th value from start up to stop−1. A negative step counts downward: range(10, 0, -1) yields 10, 9, … 1.
range(5) # 0 1 2 3 4 range(2, 7) # 2 3 4 5 6 range(0, 10, 3) # 0 3 6 9 range(5, 0, -1) # 5 4 3 2 1
⚠️ The stop value is exclusive — range(1, 5) gives 1, 2, 3, 4 — not 5. This catches many beginners off guard.
range supports len(), membership tests (7 in range(10)), and slicing — all in O(1).
Wrap with list() only when you genuinely need a list: list(range(5)) → [0, 1, 2, 3, 4].
Mental model
break is an emergency exit from the loop. continue is a skip-this-item signal. The loop else is a "we finished without finding it" notification — it only fires when break was never reached.
The Answer
break terminates the innermost loop immediately and jumps past it. continue skips the remaining statements in the current iteration and goes back to the loop header to check the condition (or move to the next item). The else clause attached to a for or while loop runs only if the loop completed normally — i.e., it was never exited via break. This makes the loop else ideal for "search and report not found" patterns without needing a separate flag variable.
break exits only the loop it is directly inside — nested loops require a flag or restructuring to break multiple levels.
The loop else is Python-specific — developers from other languages are often surprised by it.
Mental model
Resist the urge to use range(len(...)) — it obscures intent. Python gives you enumerate() precisely for this job, and it's the idiomatic choice interviewers expect to see.
The Answer
Use enumerate(iterable, start=0). It yields (index, value) tuples, so you can unpack both in the loop header:
colors = ["red", "green", "blue"] # Idiomatic — use enumerate for i, color in enumerate(colors): print(f"{i}: {color}") # 0: red, 1: green, 2: blue # Start counting from 1 instead of 0 for i, color in enumerate(colors, start=1): print(f"{i}. {color}") # 1. red, 2. green, 3. blue # Avoid this — range(len()) is verbose and fragile for i in range(len(colors)): print(colors[i]) # works but not idiomatic
enumerate() works on any iterable, not just lists: strings, tuples, file lines, etc.
To loop two lists together, use zip(): for a, b in zip(list1, list2): — stops at the shorter list.
Mental model
Modifying a list mid-iteration is like pulling planks from a bridge while walking across it — the iterator loses its footing and items get skipped or cause errors.
The Answer
Removing or inserting items into a list while iterating over it shifts indices, causing items to be silently skipped or triggering an IndexError. The safe alternatives are: iterate over a copy of the list (for item in my_list[:]:), or build a new list with a list comprehension ([x for x in my_list if condition]).
# ❌ Buggy — items get skipped nums = [1, 2, 3, 4, 5] for n in nums: if n % 2 == 0: nums.remove(n) # shifts indices → 3 survives unscanned print(nums) # [1, 3, 5] ← wrong! 3 survived by luck # ✅ Safe — iterate over a copy for n in nums[:]: if n % 2 == 0: nums.remove(n) # ✅ Idiomatic — list comprehension nums = [n for n in nums if n % 2 != 0]
⚠️ The same problem applies to dictionaries — never add or remove keys from a dict inside a for key in dict: loop. Iterate over list(dict.keys()) instead.
Prefer list comprehensions for filtering — they are faster, safer, and more Pythonic than in-place deletion inside a loop.
If you must mutate in-place, loop in reverse: for i in range(len(lst)-1, -1, -1): and delete by index.

if / elif / else
Bedingte Verzweigung — Entscheidungen im Code treffen

Verzweigung mit if, elif und else

Python wertet die erste Bedingung aus, die truthy ist, und führt diesen Block aus. Alle anderen Zweige werden übersprungen. Einrückung (4 Leerzeichen) ist obligatorisch — sie ersetzt die geschweiften Klammern anderer Sprachen.

punkte = 78 if punkte >= 90: note = "Sehr gut" elif punkte >= 75: note = "Gut" # ← dieser Zweig wird ausgeführt elif punkte >= 60: note = "Befriedigend" else: note = "Nicht bestanden" print(note) # → Gut
Vergleichsoperatoren: ==, !=, <, <=, >, >=. Verwende == für Gleichheit, niemals = (das ist Zuweisung).
Logische Operatoren: and, or, not. Python verwendet Wörter, nicht && / ||.
Vergleiche können verkettet werden: 0 <= x < 100 ist gültiges Python und liest sich wie in der Mathematik.
elif und else sind optional. Ein einfaches if ohne else ist vollkommen gültig.

for-Schleifen
Über Sequenzen iterieren — die häufigste Schleife in Python

Iterieren mit for und range()

Eine for-Schleife in Python iteriert über jedes Iterable — eine Liste, einen String, eine Range, ein Tupel, ein Dictionary oder eine Datei — jeweils ein Element nach dem anderen. Ein manuelles Verwalten eines Index-Zählers ist nie nötig.

# Über eine Liste iterieren obst = ["Apfel", "Banane", "Kirsche"] for frucht in obst: print(frucht) # Apfel, dann Banane, dann Kirsche # Feste Anzahl von Iterationen mit range() for i in range(5): # 0, 1, 2, 3, 4 print(i) # range(start, stop, schritt) for n in range(2, 11, 2): # 2, 4, 6, 8, 10 print(n) # Mit Index UND Wert — enumerate() verwenden for idx, wert in enumerate(obst): print(idx, wert) # 0 Apfel, 1 Banane, 2 Kirsche
range(stop) → 0 bis stop−1. range(start, stop) → start bis stop−1. range(start, stop, schritt) → jeden schritt-ten Wert.
range ist lazy — es wird keine Liste im Speicher aufgebaut. Verwende list(range(5)), wenn du eine echte Liste brauchst.
Bevorzuge enumerate() gegenüber einer manuell verwalteten Indexvariable — es ist lesbarer und weniger fehleranfällig.
Um zwei Listen parallel zu durchlaufen, verwende zip(liste_a, liste_b).

while-Schleifen
Wiederholen, solange eine Bedingung wahr ist

Wiederholen mit while

Eine while-Schleife führt ihren Rumpf aus, solange ihre Bedingung truthy ist. Verwende sie, wenn du im Voraus nicht weißt, wie viele Iterationen benötigt werden — zum Beispiel beim Einlesen von Benutzereingaben bis sie gültig sind, oder beim Pollen bis eine Ressource bereit ist.

# Klassischer Countdown zaehler = 5 while zaehler > 0: print(zaehler) zaehler -= 1 print("Fertig!") # → 5 4 3 2 1 Fertig! # Benutzereingabe validieren while True: # Endlosschleife — mit break beendet eingabe = input("Gib eine positive Zahl ein: ") if eingabe.isdigit() and int(eingabe) > 0: zahl = int(eingabe) break print("Ungültig — erneut versuchen.")
Stelle immer sicher, dass die Schleifenbedingung irgendwann False wird, oder verwende break — eine Endlosschleife friert dein Programm ein.
Das Muster while True: … break ist idiomatisches Python für "solange fragen, bis eine gültige Eingabe vorliegt".
Wenn du eine while-Schleife mit einer Zählvariable schreibst, ist eine for … range()-Schleife meist klarer.

break, continue & Schleifen-else
Feinsteuerung innerhalb von Schleifen

Schleifen steuern mit break, continue und else

break beendet die Schleife sofort. continue überspringt den Rest der aktuellen Iteration und springt zur nächsten. Die optionale else-Klausel einer Schleife wird nur ausgeführt, wenn die Schleife nicht durch break beendet wurde.

# break — stoppe, sobald der Zielwert gefunden wird zahlen = [3, 7, 2, 9, 4] for n in zahlen: if n == 9: print("9 gefunden!") break # beendet Schleife; else-Klausel läuft NICHT # continue — gerade Zahlen überspringen, nur ungerade ausgeben for n in range(1, 10): if n % 2 == 0: continue # zur nächsten Iteration springen print(n) # → 1 3 5 7 9 # Schleifen-else — läuft nur, wenn break nie ausgelöst wurde for n in zahlen: if n == 99: print("99 gefunden!") break else: print("99 nicht in der Liste") # ← dies wird ausgegeben
break beendet nur die innerste Schleife. Um aus verschachtelten Schleifen auszubrechen, verwende eine Flag-Variable oder lagere in eine Funktion aus und nutze return.
continue ist nützlich zum Filtern: Elemente, die dich nicht interessieren, überspringen und den Hauptpfad unnested halten.
Das Schleifen-else ist Python-spezifisch und überrascht oft Entwickler aus anderen Sprachen. Nutze es für "Suchen und Melden, wenn nicht gefunden"-Muster.

Schnellreferenz

Schlüsselwort Wo verwendbar Wirkung
break for oder while Beendet die innerste Schleife sofort
continue for oder while Springt zur nächsten Iteration
else (Schleife) Nach for oder while Läuft nur, wenn die Schleife ohne break endete
pass Überall wo eine Anweisung erwartet wird Tut nichts — Platzhalter für leere Blöcke

Übungsfragen
Typische Interview- und Selbstkontrollfragen zu Control Flow
Mentales Modell
Python verlangt keinen booleschen Wert — jeder Wert kann als Bedingung stehen. Das Kennen der falsy-Menge ermöglicht sauberere, idiomatische Guards.
Die Antwort
Python konvertiert die Bedingung mithilfe der __bool__-Methode des Objekts in einen Boolean. Die falschen Werte (falsy) sind: False, 0, 0.0, 0j, None, "", [], {}, () und set(). Alle anderen Werte sind truthy. Das bedeutet: if meine_liste: ist die idiomatische Art zu prüfen, ob eine Liste nicht leer ist — kein len(meine_liste) > 0 nötig.
⚠️ if wert == False: trifft auch 0 und 0.0. Verwende if wert is False:, wenn du gezielt den booleschen Singleton brauchst.
Lerne die falsy-Menge auswendig: None, 0, leerer String, leere Container.
Bevorzuge if elemente: gegenüber if len(elemente) != 0:.
Mentales Modell
Eine for-Schleife sagt "tue dies für jedes Element einer Sequenz". Eine while-Schleife sagt "mache das weiter, bis sich eine Bedingung ändert". Die Schlüsselfrage: Kennst du die Sammlung im Voraus?
Die Antwort
Verwende eine for-Schleife, wenn du ein bekanntes Iterable hast (Liste, Range, String, Datei) — sie ist übersichtlicher und eliminiert Off-by-one-Fehler. Verwende eine while-Schleife, wenn die Anzahl der Iterationen von einer zur Laufzeit ausgewerteten Bedingung abhängt, zum Beispiel beim Warten auf gültige Benutzereingaben, beim Pollen eines Sensors oder beim Lesen eines Streams bis EOF. In der Praxis sind for-Schleifen in Python weitaus häufiger.
for-Schleife → bekannte Sammlung; while-Schleife → unbekannte Dauer.
Eine while-Schleife mit Zähler sollte fast immer ein for i in range(...): sein.
Mentales Modell
Stell dir range als eine lazy arithmetische Sequenz vor — sie merkt sich Start, Stop und Schritt und berechnet jeden Wert bei Bedarf. Sie hält nie alle Zahlen gleichzeitig im Speicher.
Die Antwort
range() gibt ein range-Objekt zurück (eine lazy Sequenz), keine Liste. Die drei Formen sind: range(stop) → 0 bis stop−1; range(start, stop) → start bis stop−1; range(start, stop, schritt) → jeden schritt-ten Wert von start bis stop−1. Ein negativer Schritt zählt rückwärts: range(10, 0, -1) ergibt 10, 9, … 1.
range(5) # 0 1 2 3 4 range(2, 7) # 2 3 4 5 6 range(0, 10, 3) # 0 3 6 9 range(5, 0, -1) # 5 4 3 2 1
⚠️ Der Stop-Wert ist exklusiv — range(1, 5) ergibt 1, 2, 3, 4 — nicht 5. Das überrascht viele Anfänger.
range unterstützt len(), Mitgliedschaftstests (7 in range(10)) und Slicing — alles in O(1).
Wrapping mit list() nur wenn wirklich eine Liste benötigt wird: list(range(5)) → [0, 1, 2, 3, 4].
Mentales Modell
break ist ein Notausgang aus der Schleife. continue ist ein "Dieses Element überspringen"-Signal. Das Schleifen-else ist eine "Wir sind fertig, ohne es zu finden"-Meldung — es feuert nur, wenn break nie erreicht wurde.
Die Antwort
break beendet die innerste Schleife sofort und springt dahinter. continue überspringt die restlichen Anweisungen der aktuellen Iteration und geht zurück zum Schleifenkopf. Die else-Klausel einer for- oder while-Schleife läuft nur, wenn die Schleife normal beendet wurde — also nie durch break verlassen wurde. Das macht das Schleifen-else ideal für "Suche und melde, wenn nicht gefunden"-Muster ohne separate Flag-Variable.
break beendet nur die direkt umgebende Schleife — bei verschachtelten Schleifen braucht man eine Flag oder Umstrukturierung.
Das Schleifen-else ist Python-spezifisch — Entwickler aus anderen Sprachen sind oft überrascht davon.
Mentales Modell
Widerstehe dem Drang, range(len(...)) zu verwenden — das verschleiert die Absicht. Python bietet enumerate() genau für diesen Zweck, und das ist die idiomatische Wahl.
Die Antwort
Verwende enumerate(iterable, start=0). Es liefert (index, wert)-Tupel, die du im Schleifenkopf entpacken kannst:
farben = ["rot", "grün", "blau"] # Idiomatisch — enumerate verwenden for i, farbe in enumerate(farben): print(f"{i}: {farbe}") # 0: rot, 1: grün, 2: blau # Ab 1 zählen statt 0 for i, farbe in enumerate(farben, start=1): print(f"{i}. {farbe}") # 1. rot, 2. grün, 3. blau # Vermeide dies — range(len()) ist umständlich for i in range(len(farben)): print(farben[i]) # funktioniert, aber nicht idiomatisch
enumerate() funktioniert mit jedem Iterable, nicht nur mit Listen: Strings, Tupel, Dateieinträge usw.
Um zwei Listen parallel zu durchlaufen, verwende zip(): for a, b in zip(liste1, liste2): — stoppt bei der kürzeren Liste.
Mentales Modell
Eine Liste während der Iteration zu verändern ist wie Bretter aus einer Brücke zu reißen, während man darüber läuft — der Iterator verliert den Boden und Elemente werden übersprungen oder es entstehen Fehler.
Die Antwort
Elemente während der Iteration zu entfernen oder einzufügen verschiebt Indizes, wodurch Elemente lautlos übersprungen werden oder ein IndexError entsteht. Die sicheren Alternativen sind: über eine Kopie der Liste iterieren (for item in meine_liste[:]:) oder eine neue Liste per List Comprehension erstellen ([x for x in meine_liste if bedingung]).
# ❌ Fehlerhaft — Elemente werden übersprungen zahlen = [1, 2, 3, 4, 5] for n in zahlen: if n % 2 == 0: zahlen.remove(n) # verschiebt Indizes → 3 wird nicht geprüft print(zahlen) # [1, 3, 5] ← falsch! # ✅ Sicher — über eine Kopie iterieren for n in zahlen[:]: if n % 2 == 0: zahlen.remove(n) # ✅ Idiomatisch — List Comprehension zahlen = [n for n in zahlen if n % 2 != 0]
⚠️ Das gleiche Problem gilt für Dictionaries — niemals Schlüssel in einer for key in dict:-Schleife hinzufügen oder entfernen. Stattdessen über list(dict.keys()) iterieren.
Bevorzuge List Comprehensions zum Filtern — sie sind schneller, sicherer und pythonischer als In-Place-Löschung in einer Schleife.
Falls In-Place-Mutation nötig ist, iteriere rückwärts: for i in range(len(lst)-1, -1, -1): und lösche per Index.
