Defining Functions
def, parameters, return values, and docstrings

How to define and call a function

A function is a named, reusable block of code defined with the def keyword. The indented body runs only when the function is called. A return statement sends a value back to the caller — a function with no return implicitly returns None.

def greet(name): """Return a personalised greeting string.""" # docstring return f"Hello, {name}!" message = greet("Alice") # call the function print(message) # → Hello, Alice! # Function with no return value def log(text): print(f"[LOG] {text}") result = log("Starting up") print(result) # → None (implicit return)
Docstrings — a string literal on the first line of the body — are the standard way to document what a function does. Access them with help(greet) or greet.__doc__.
A function can return multiple values by returning a tuple: return x, y — the caller unpacks it with a, b = func().
Functions are first-class objects in Python — you can assign them to variables, pass them as arguments, and store them in lists.

Positional & Keyword Arguments
How Python matches arguments to parameters — and the default value trap

Positional, keyword, and default arguments

Positional arguments are matched to parameters by order. Keyword arguments are matched by name and can be passed in any order. Default values make a parameter optional — callers who omit it get the default.

def describe(name, age, city="Unknown"): return f"{name}, {age}, from {city}" # Positional — order matters describe("Alice", 30, "Berlin") # → Alice, 30, from Berlin # Keyword — order doesn't matter describe(age=25, name="Bob") # → Bob, 25, from Unknown # Mix: positional first, then keyword describe("Carol", 28, city="Paris") # → Carol, 28, from Paris
Parameters with defaults must come after parameters without defaults in the function signature.
You can force callers to use keyword arguments by placing a bare * in the signature: def func(a, *, b): — b must then be passed as func(1, b=2).
Keyword arguments improve readability at the call site — connect(host="localhost", port=5432) is far clearer than connect("localhost", 5432).

*args and **kwargs — accepting any number of arguments

*args collects extra positional arguments into a tuple. **kwargs collects extra keyword arguments into a dictionary. Both are often used to write flexible wrapper functions.

def total(*args): """Sum any number of values.""" return sum(args) total(1, 2, 3) # → 6 total(10, 20) # → 30 def show_info(**kwargs): """Print any named fields.""" for key, value in kwargs.items(): print(f" {key}: {value}") show_info(name="Alice", role="admin", level=3) # → name: Alice # → role: admin # → level: 3
The names args and kwargs are conventions, not keywords — what matters is the * and ** prefix.
You can also use * and ** at the call site to unpack a list or dict into separate arguments: func(*my_list), func(**my_dict).

Scope & the LEGB Rule
How Python looks up names — Local → Enclosing → Global → Built-in

Name resolution order: LEGB

When Python encounters a name, it searches four scopes in order until it finds a match or raises a NameError. Understanding this order prevents subtle bugs where a local variable shadows a global one, or a built-in function is accidentally overwritten.

x = "global" # Global scope def outer(): x = "enclosing" # Enclosing scope def inner(): x = "local" # Local scope print(x) # → local (L wins) inner() print(x) # → enclosing outer() print(x) # → global # Built-in scope — names like len, print, range live here print(len("hello")) # → 5 (found in Built-in)
Letter Scope Where it lives
L Local Inside the current function call
E Enclosing Any outer functions that wrap the current one
G Global Top-level of the current module / file
B Built-in Python's standard built-ins (len, print, range, …)
Assigning to a name inside a function always creates a local variable — it does not modify the global by default.
Use global x inside a function to explicitly write to a global variable — but prefer returning values instead; global makes code harder to test.
Never shadow built-ins: naming a variable list, str, or input hides the built-in for the rest of that scope.

Practice Questions
Typical interview and self-check questions on functions
Mental model
A function is a named recipe: def writes the recipe down, calling it cooks the dish. return is how the kitchen hands the finished dish back to whoever ordered it.
The Answer
Use the def keyword followed by the function name, parenthesised parameters, and a colon. The indented body is the function's code. return immediately exits the function and sends a value back to the caller. A function that reaches the end without a return statement implicitly returns None. A single function can have multiple return statements in different branches.
def add(a, b): return a + b # exits immediately, value goes to caller result = add(3, 4) # result = 7 def nothing(): pass # no return → implicitly returns None print(nothing()) # → None
Return multiple values as a tuple: return min_val, max_val — unpack at the call site: lo, hi = bounds(data).
Functions are objects — type(add) returns <class 'function'>.
Mental model
Positional arguments fill seats by order — first in, first matched. Keyword arguments fill seats by name tag — you can arrive in any order as long as your name is on the list. Defaults are the seat that's pre-occupied when nobody shows up.
The Answer
Positional arguments are matched to parameters left to right by position. Keyword arguments use name=value syntax and can appear in any order. Default values are specified in the function signature with param=default and are used when the caller omits that argument. Parameters with defaults must follow parameters without them.
def connect(host, port=5432, timeout=30): print(f"Connecting to {host}:{port} (timeout={timeout}s)") connect("db.example.com") # uses both defaults connect("db.example.com", 3306) # positional override connect("db.example.com", timeout=10) # keyword override, port stays 5432
⚠️ Once you use a keyword argument at the call site, all subsequent arguments must also be keyword arguments — you cannot switch back to positional.
Keyword arguments make call sites self-documenting — send(to="alice", subject="Hi") is instantly readable.
Default values are evaluated once at function definition time — not each time the function is called. This is the source of the mutable-default trap (see Q04).
Mental model
Think of Python searching outwards through nested rooms. It checks your own room (Local) first, then any hallways you're in (Enclosing), then the building foyer (Global), and finally the city directory (Built-in). It stops at the first room that has what it needs.
The Answer
LEGB stands for Local → Enclosing → Global → Built-in. When Python looks up a name, it searches these four scopes in order and uses the first match it finds. If no match is found anywhere, a NameError is raised. Assigning a name inside a function always creates a local variable — it does not modify the global version unless you explicitly declare global name.
count = 0 # Global def increment(): global count # tells Python: use the global 'count' count += 1 increment() print(count) # → 1 # Without 'global', count += 1 would raise UnboundLocalError # because Python sees the assignment and creates a local 'count' # that hasn't been assigned yet before being read.
⚠️ Avoid global in production code. Prefer passing values as arguments and returning results — it keeps functions predictable and testable.
Shadowing a built-in is legal but dangerous: list = [1, 2, 3] hides the built-in list type for the rest of that scope.
Inner functions can read variables from their enclosing scope without any keyword — but to write to them, they need nonlocal name.
Mental model
The default value is created once when Python reads the def statement — not freshly each time you call the function. A mutable default like [] is a single shared object that persists across calls.
The Answer
Default argument values are evaluated exactly once at function definition time. If the default is a mutable object (list, dict, set), all calls that use the default share the same object. Mutating it in one call changes it for all future calls — a notoriously surprising bug. The fix is to use None as the default and create the mutable object inside the function body.
# ❌ Bug — all calls share the same list def append_item(item, result=[]): result.append(item) return result append_item(1) # → [1] append_item(2) # → [1, 2] ← unexpected! append_item(3) # → [1, 2, 3] ← the list was never reset # ✅ Fix — use None as sentinel, create fresh list each call def append_item(item, result=None): if result is None: result = [] result.append(item) return result append_item(1) # → [1] append_item(2) # → [2] ← correct, fresh list each time
⚠️ This trap applies to any mutable default: [], {}, set(), and even custom objects. Immutable defaults like 0, "", and None are safe.
The None-sentinel pattern is the standard Python idiom for optional mutable parameters.
You can inspect a function's defaults: append_item.__defaults__ — useful for debugging this trap.
Mental model
*args is a catch-all bag for extra positional arguments. **kwargs is a catch-all bag for extra keyword arguments. Both are used when you want a function to accept an open-ended number of inputs.
The Answer
*args collects any number of extra positional arguments into a tuple. **kwargs collects any number of extra keyword arguments into a dict. Common use cases include building flexible wrapper or decorator functions, forwarding arguments to another function, and creating APIs that accept optional named fields.
def log(level, *args, **kwargs): print(f"[{level}]", *args) if kwargs: print(" extra:", kwargs) log("INFO", "Server started", port=8080) # → [INFO] Server started # → extra: {'port': 8080} # Unpacking at the call site nums = [1, 2, 3] print(*nums) # same as print(1, 2, 3) settings = {"sep": " | ", "end": "\n"} print("a", "b", **settings) # same as print("a", "b", sep=" | ", end="\n")
Signature order rule: def f(normal, *args, keyword_only, **kwargs): — regular params first, then *args, then keyword-only params, then **kwargs.
The names args and kwargs are conventions, not keywords. *numbers or **options work equally well.
Mental model
A lambda is a throwaway function — useful when you need a simple expression in a place that expects a callable but don't want to pollute the namespace with a named function.
The Answer
A lambda creates an anonymous function with a single expression as its body. It is syntactic sugar for a def that only has a return statement. Lambdas are best used inline — most commonly as the key argument to sorted(), min(), and max(), or as short callbacks. If the logic is more than one expression, or if you need to reuse it, use a named def instead.
# Lambda syntax: lambda parameters: expression square = lambda x: x ** 2 square(5) # → 25 # Typical use — inline sort key people = [("Alice", 30), ("Bob", 25), ("Carol", 35)] sorted(people, key=lambda p: p[1]) # → [('Bob', 25), ('Alice', 30), ('Carol', 35)] # Equivalent named function — prefer this if reused def by_age(person): return person[1] sorted(people, key=by_age)
⚠️ Never assign a lambda to a variable and reuse it — that is exactly what def is for. PEP 8 discourages f = lambda x: x + 1; use def f(x): return x + 1 instead.
Lambdas can take multiple parameters: lambda x, y: x + y.
Lambdas cannot contain statements — no if/else blocks, no for loops, no return. Use def for anything beyond a single expression.

Funktionen definieren
def, Parameter, Rückgabewerte und Docstrings

Wie man eine Funktion definiert und aufruft

Eine Funktion ist ein benannter, wiederverwendbarer Codeblock, der mit dem Schlüsselwort def definiert wird. Der eingerückte Rumpf wird nur ausgeführt, wenn die Funktion aufgerufen wird. Eine return-Anweisung sendet einen Wert zurück an den Aufrufer — eine Funktion ohne return gibt implizit None zurück.

def begruessen(name): """Gibt einen personalisierten Begrüßungsstring zurück.""" # docstring return f"Hallo, {name}!" nachricht = begruessen("Alice") # Funktion aufrufen print(nachricht) # → Hallo, Alice! # Funktion ohne Rückgabewert def protokollieren(text): print(f"[LOG] {text}") ergebnis = protokollieren("Startet") print(ergebnis) # → None (implizite Rückgabe)
Docstrings — ein String-Literal in der ersten Zeile des Rumpfes — sind die Standardmethode zur Dokumentation einer Funktion. Abrufbar mit help(begruessen) oder begruessen.__doc__.
Eine Funktion kann mehrere Werte als Tupel zurückgeben: return x, y — der Aufrufer entpackt sie mit a, b = func().
Funktionen sind erstklassige Objekte in Python — sie können Variablen zugewiesen, als Argumente übergeben und in Listen gespeichert werden.

Positions- & Schlüsselwortargumente
Wie Python Argumente Parametern zuordnet — und die Standardwert-Falle

Positions-, Schlüsselwort- und Standardargumente

Positionsargumente werden Parametern der Reihe nach zugeordnet. Schlüsselwortargumente werden namentlich zugeordnet und können in beliebiger Reihenfolge übergeben werden. Standardwerte machen einen Parameter optional — wer ihn weglässt, erhält den Standard.

def beschreiben(name, alter, stadt="Unbekannt"): return f"{name}, {alter}, aus {stadt}" # Positional — Reihenfolge ist wichtig beschreiben("Alice", 30, "Berlin") # → Alice, 30, aus Berlin # Schlüsselwort — Reihenfolge egal beschreiben(alter=25, name="Bob") # → Bob, 25, aus Unbekannt # Mix: zuerst positional, dann Schlüsselwort beschreiben("Carol", 28, stadt="Paris") # → Carol, 28, aus Paris
Parameter mit Standardwerten müssen nach Parametern ohne Standardwerte stehen.
Durch ein einzelnes * in der Signatur kannst du erzwingen, dass nachfolgende Parameter als Schlüsselwortargumente übergeben werden müssen: def func(a, *, b):.
Schlüsselwortargumente verbessern die Lesbarkeit — verbinden(host="localhost", port=5432) ist sofort verständlich.

*args und **kwargs — beliebig viele Argumente

*args sammelt zusätzliche Positionsargumente in einem Tupel. **kwargs sammelt zusätzliche Schlüsselwortargumente in einem Dictionary. Beides wird häufig für flexible Wrapper-Funktionen genutzt.

def summe(*args): """Addiert beliebig viele Werte.""" return sum(args) summe(1, 2, 3) # → 6 summe(10, 20) # → 30 def info_ausgeben(**kwargs): """Gibt beliebige benannte Felder aus.""" for schluessel, wert in kwargs.items(): print(f" {schluessel}: {wert}") info_ausgeben(name="Alice", rolle="Admin", stufe=3) # → name: Alice # → rolle: Admin # → stufe: 3
Die Namen args und kwargs sind Konventionen, keine Schlüsselwörter — entscheidend sind nur die Präfixe * und **.
* und ** können auch beim Aufruf genutzt werden, um eine Liste oder ein Dict zu entpacken: func(*meine_liste), func(**mein_dict).

Scope & die LEGB-Regel
Wie Python Namen auflöst — Lokal → Einschließend → Global → Eingebaut

Namensauflösungsreihenfolge: LEGB

Wenn Python auf einen Namen trifft, durchsucht es vier Scopes der Reihe nach, bis es eine Übereinstimmung findet oder einen NameError auslöst. Das Verstehen dieser Reihenfolge verhindert subtile Fehler, bei denen eine lokale Variable eine globale verdeckt oder ein eingebautes Konstrukt versehentlich überschrieben wird.

x = "global" # Globaler Scope def aeussere(): x = "einschliessend" # Einschließender Scope def innere(): x = "lokal" # Lokaler Scope print(x) # → lokal (L gewinnt) innere() print(x) # → einschliessend aeussere() print(x) # → global # Eingebauter Scope — len, print, range usw. leben hier print(len("Hallo")) # → 5 (im Built-in gefunden)
Buchstabe Scope Wo er lebt
L Lokal (Local) Innerhalb des aktuellen Funktionsaufrufs
E Einschließend (Enclosing) Äußere Funktionen, die die aktuelle umgeben
G Global Oberste Ebene des aktuellen Moduls / der Datei
B Eingebaut (Built-in) Pythons Standard-Built-ins (len, print, range, …)
Eine Zuweisung innerhalb einer Funktion erstellt immer eine lokale Variable — sie verändert nicht den globalen Wert, außer mit global.
Verwende global x in einer Funktion, um explizit eine globale Variable zu beschreiben — besser aber: Werte als Argumente übergeben und als Rückgabewerte erhalten.
Eingebaute Namen niemals überschatten: eine Variable namens list, str oder input verbirgt das eingebaute Konstrukt für den Rest des Scopes.

Übungsfragen
Typische Interview- und Selbstkontrollfragen zu Funktionen
Mentales Modell
Eine Funktion ist ein benanntes Rezept: def schreibt das Rezept auf, der Aufruf kocht das Gericht. return ist die Art, wie die Küche das fertige Gericht zurück an den Besteller übergibt.
Die Antwort
Das Schlüsselwort def wird gefolgt vom Funktionsnamen, geklammerten Parametern und einem Doppelpunkt. Der eingerückte Rumpf ist der Code der Funktion. return beendet die Funktion sofort und gibt einen Wert an den Aufrufer zurück. Eine Funktion, die das Ende ohne return-Anweisung erreicht, gibt implizit None zurück.
def addieren(a, b): return a + b # beendet sofort, Wert geht an Aufrufer ergebnis = addieren(3, 4) # ergebnis = 7 def nichts(): pass # kein return → implizit None print(nichts()) # → None
Mehrere Werte als Tupel zurückgeben: return min_val, max_val — beim Aufruf entpacken: lo, hi = grenzen(daten).
Funktionen sind Objekte — type(addieren) ergibt <class 'function'>.
Mentales Modell
Positionsargumente belegen Plätze nach Reihenfolge — erste rein, erste zugeordnet. Schlüsselwortargumente belegen Plätze nach Namensschild — die Reihenfolge spielt keine Rolle. Standardwerte sind Plätze, die vorbesetzt sind, wenn niemand erscheint.
Die Antwort
Positionsargumente werden Parametern von links nach rechts nach Position zugeordnet. Schlüsselwortargumente verwenden die Syntax name=wert und können in beliebiger Reihenfolge angegeben werden. Standardwerte werden in der Funktionssignatur mit param=standard angegeben und genutzt, wenn der Aufrufer das Argument weglässt.
def verbinden(host, port=5432, timeout=30): print(f"Verbinde mit {host}:{port} (Timeout={timeout}s)") verbinden("db.example.com") # beide Standards verbinden("db.example.com", 3306) # positionale Überschreibung verbinden("db.example.com", timeout=10) # Schlüsselwort, port bleibt 5432
⚠️ Sobald beim Aufruf ein Schlüsselwortargument verwendet wird, müssen alle nachfolgenden Argumente ebenfalls als Schlüsselwortargumente übergeben werden.
Schlüsselwortargumente machen den Aufruf selbstdokumentierend: senden(an="alice", betreff="Hi") ist sofort lesbar.
Standardwerte werden einmal bei der Funktionsdefinition ausgewertet — nicht bei jedem Aufruf. Daraus entsteht die Mutable-Default-Falle (siehe F04).
Mentales Modell
Stell dir Python vor, das konzentrisch nach außen durch verschachtelte Räume sucht: erst dein eigener Raum (Lokal), dann der Flur (Einschließend), dann der Gebäudeeingang (Global), schließlich das Stadtverzeichnis (Eingebaut). Es stoppt beim ersten Raum, der das Gesuchte enthält.
Die Antwort
LEGB steht für Lokal → Einschl. (Enclosing) → Global → Built-in. Python durchsucht diese vier Scopes in dieser Reihenfolge und nimmt den ersten Treffer. Gibt es keinen, wird ein NameError ausgelöst. Eine Zuweisung innerhalb einer Funktion erstellt immer eine lokale Variable — um die globale zu beschreiben, braucht man explizit global name.
zaehler = 0 # Global def erhoehen(): global zaehler # Python: nutze das globale 'zaehler' zaehler += 1 erhoehen() print(zaehler) # → 1 # Ohne 'global' würde zaehler += 1 einen UnboundLocalError auslösen
⚠️ Vermeide global in Produktionscode. Besser: Werte als Argumente übergeben und als Rückgabewerte erhalten — das macht Funktionen vorhersagbar und testbar.
Ein eingebautes Konstrukt zu überschatten ist legal aber gefährlich: list = [1, 2, 3] verbirgt den eingebauten Typ list für den Rest des Scopes.
Innere Funktionen können Variablen aus dem einschließenden Scope lesen — zum Schreiben benötigen sie nonlocal name.
Mentales Modell
Der Standardwert wird einmal erstellt, wenn Python die def-Anweisung liest — nicht neu bei jedem Aufruf. Ein veränderliches Objekt wie [] ist ein einziges geteiltes Objekt, das über alle Aufrufe hinweg weiterbesteht.
Die Antwort
Standardargumentwerte werden genau einmal bei der Funktionsdefinition ausgewertet. Ist das Standardobjekt veränderlich (Liste, Dict, Set), teilen sich alle Aufrufe, die den Standard nutzen, dasselbe Objekt. Eine Mutation in einem Aufruf verändert es für alle zukünftigen Aufrufe. Die Lösung: None als Standard verwenden und das veränderliche Objekt im Funktionsrumpf anlegen.
# ❌ Fehler — alle Aufrufe teilen dieselbe Liste def element_hinzufuegen(item, ergebnis=[]): ergebnis.append(item) return ergebnis element_hinzufuegen(1) # → [1] element_hinzufuegen(2) # → [1, 2] ← unerwartet! # ✅ Korrekt — None als Sentinel verwenden def element_hinzufuegen(item, ergebnis=None): if ergebnis is None: ergebnis = [] ergebnis.append(item) return ergebnis element_hinzufuegen(1) # → [1] element_hinzufuegen(2) # → [2] ← korrekt
⚠️ Diese Falle gilt für alle veränderlichen Standards: [], {}, set() und eigene Objekte. Unveränderliche Standards wie 0, "" und None sind sicher.
Das None-Sentinel-Muster ist das Standard-Python-Idiom für optionale veränderliche Parameter.
Standardwerte einer Funktion können inspiziert werden: func.__defaults__.
Mentales Modell
*args ist eine Auffang-Tasche für zusätzliche Positionsargumente. **kwargs ist eine Auffang-Tasche für zusätzliche Schlüsselwortargumente. Beide werden verwendet, wenn eine Funktion eine offene Anzahl von Eingaben akzeptieren soll.
Die Antwort
*args sammelt beliebig viele zusätzliche Positionsargumente in einem Tupel. **kwargs sammelt beliebig viele zusätzliche Schlüsselwortargumente in einem Dict. Typische Anwendungsfälle sind flexible Wrapper-Funktionen, das Weiterleiten von Argumenten an andere Funktionen und APIs, die optionale benannte Felder akzeptieren.
def protokoll(stufe, *args, **kwargs): print(f"[{stufe}]", *args) if kwargs: print(" extra:", kwargs) protokoll("INFO", "Server gestartet", port=8080) # → [INFO] Server gestartet # → extra: {'port': 8080} # Entpacken beim Aufruf zahlen = [1, 2, 3] print(*zahlen) # wie print(1, 2, 3) optionen = {"sep": " | ", "end": "\n"} print("a", "b", **optionen)
Reihenfolge in der Signatur: def f(normal, *args, schluessel, **kwargs): — normale Params zuerst, dann *args, dann schlüsselwortbasierte Params, dann **kwargs.
args und kwargs sind Konventionen, keine Schlüsselwörter — *zahlen oder **optionen funktionieren genauso.
Mentales Modell
Ein Lambda ist eine Wegwerf-Funktion — nützlich, wenn an einer Stelle, die ein Callable erwartet, eine einfache Logik benötigt wird, ohne den Namensraum mit einer benannten Funktion zu überladen.
Die Antwort
Ein lambda erzeugt eine anonyme Funktion mit einem einzigen Ausdruck als Rumpf. Es ist syntaktischer Zucker für ein def mit nur einer return-Anweisung. Lambdas eignen sich am besten inline — häufig als key-Argument für sorted(), min() und max(). Für komplexere Logik oder Wiederverwendung ist ein benanntes def vorzuziehen.
# Lambda-Syntax: lambda parameter: ausdruck quadrat = lambda x: x ** 2 quadrat(5) # → 25 # Typischer Einsatz — Sortierschlüssel inline personen = [("Alice", 30), ("Bob", 25), ("Carol", 35)] sorted(personen, key=lambda p: p[1]) # → [('Bob', 25), ('Alice', 30), ('Carol', 35)] # Äquivalente benannte Funktion — bevorzugen, wenn wiederverwendet def nach_alter(person): return person[1] sorted(personen, key=nach_alter)
⚠️ Weise ein Lambda niemals einer Variablen zu und verwende es mehrfach — dafür gibt es def. PEP 8 rät von f = lambda x: x + 1 ab; stattdessen def f(x): return x + 1.
Lambdas können mehrere Parameter haben: lambda x, y: x + y.
Lambdas dürfen keine Anweisungen enthalten — kein if/else-Block, keine for-Schleife, kein return. Für alles außer einem einzigen Ausdruck def verwenden.
