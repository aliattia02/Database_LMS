Lists
Ordered, mutable sequences — Python's most versatile collection

Creating, indexing, and slicing lists

A list is an ordered, mutable sequence that can hold items of any type, including mixed types. It is defined with square brackets. Items are accessed by zero-based index; negative indices count from the end.

fruits = ["apple", "banana", "cherry", "date"] # Indexing fruits[0] # → "apple" (first) fruits[-1] # → "date" (last) fruits[-2] # → "cherry" (second from end) # Slicing [start : stop : step] — stop is exclusive fruits[1:3] # → ["banana", "cherry"] fruits[:2] # → ["apple", "banana"] fruits[::2] # → ["apple", "cherry"] (every 2nd) fruits[::-1] # → ["date", "cherry", "banana", "apple"] (reversed)

Mutating lists

nums = [3, 1, 4, 1, 5] nums.append(9) # add to end → [3, 1, 4, 1, 5, 9] nums.insert(0, 0) # insert at index → [0, 3, 1, 4, 1, 5, 9] nums.remove(1) # removes FIRST occurrence of 1 nums.pop() # removes & returns last item → 9 nums.pop(0) # removes & returns item at index 0 nums.sort() # sorts in place (returns None) nums.reverse() # reverses in place (returns None) # Non-mutating alternatives sorted(nums) # returns a new sorted list, nums unchanged nums + [10, 11] # concatenation — new list
append() is O(1); insert(0, x) is O(n) — inserting at the front shifts every element. Use collections.deque if you need fast front insertions.
sort() and sorted() accept a key function: sorted(words, key=str.lower).
in operator checks membership: "apple" in fruits → True. On a list this is O(n) — use a set when you need frequent membership tests.

List comprehensions

A list comprehension builds a new list by applying an expression to each item in an iterable, optionally filtered by a condition. It is faster and more readable than an equivalent for loop that calls append().

# [expression for item in iterable if condition] squares = [x**2 for x in range(1, 6)] # → [1, 4, 9, 16, 25] evens = [x for x in range(10) if x % 2 == 0] # → [0, 2, 4, 6, 8] upper = [s.upper() for s in ["a", "b", "c"]] # → ["A", "B", "C"] # Nested comprehension — flatten a 2D list matrix = [[1, 2], [3, 4], [5, 6]] flat = [n for row in matrix for n in row] # → [1, 2, 3, 4, 5, 6]

Tuples
Ordered, immutable sequences — for fixed records and safe return values

When to use a tuple instead of a list

A tuple looks like a list but uses parentheses and is immutable — its contents cannot be changed after creation. This makes tuples useful for data that should not be modified, as dictionary keys, and as lightweight records with a fixed number of fields.

point = (3, 7) # 2D coordinate — semantically fixed rgb = (255, 128, 0) # colour channel — three distinct roles # Indexing works the same as lists point[0] # → 3 # Unpacking — assign each element to a name x, y = point lat, lon = (52.52, 13.40) # Swap variables without a temp variable a, b = 1, 2 a, b = b, a # → a=2, b=1 # Single-element tuple requires a trailing comma singleton = (42,) # NOT (42) — that's just an int in parentheses # Tuples as dictionary keys (lists cannot be keys — they're mutable) grid = {(0, 0): "start", (3, 3): "end"}
Tuples are slightly more memory-efficient and faster to iterate than lists — prefer them for data that is naturally fixed.
Functions that return multiple values actually return a tuple: return x, y is return (x, y).
Named tuples (collections.namedtuple or typing.NamedTuple) add field names to tuples, making them self-documenting without the overhead of a full class.
🗂️
Dictionaries
Key-value mappings with O(1) average lookup

Working with dictionaries

A dictionary maps immutable keys to values of any type. Lookups, insertions, and deletions are O(1) on average. Since Python 3.7 dictionaries maintain insertion order.

user = {"name": "Alice", "age": 30, "role": "admin"} # Read — two ways user["name"] # → "Alice" (KeyError if missing) user.get("email") # → None (safe, no error) user.get("email", "n/a") # → "n/a" (custom default) # Write / update user["age"] = 31 # update existing key user["city"] = "Berlin" # add new key user.update({"role": "owner", "level": 5}) # bulk update # Delete del user["city"] # raises KeyError if missing user.pop("level") # removes & returns value user.pop("level", None) # safe pop — no error if missing # Iterate for key in user: # iterate keys print(key, user[key]) for key, value in user.items(): # iterate key-value pairs for value in user.values(): # iterate values only
Always prefer .get(key, default) over bare [key] when the key might be absent — it avoids a KeyError.
collections.defaultdict automatically creates a default value for missing keys — useful for grouping or counting: defaultdict(list), defaultdict(int).
Dict comprehensions follow the same pattern as list comprehensions: {k: v for k, v in pairs}.

Valid dictionary keys

Only hashable (immutable) objects can be dictionary keys — integers, floats, strings, booleans, and tuples of hashable items. Lists, dicts, and sets are mutable and therefore not valid keys.

valid_keys = {1: "int", 3.14: "float", "x": "str", True: "bool", (0, 0): "tuple"} {[1, 2]: "list"} # ❌ TypeError: unhashable type: 'list'
🔵
Sets
Unordered collections of unique, hashable items

Sets for deduplication and membership testing

A set stores only unique items with no defined order. Like dictionary keys, set elements must be hashable. Membership testing is O(1) — far faster than scanning a list.

colours = {"red", "green", "blue", "red"} # duplicate dropped print(colours) # → {'red', 'green', 'blue'} (order varies) # Membership — O(1) vs list's O(n) "red" in colours # → True # Mutation colours.add("yellow") colours.discard("green") # safe — no error if missing colours.remove("green") # raises KeyError if missing # Deduplication nums = [1, 2, 2, 3, 3, 3] unique = list(set(nums)) # → [1, 2, 3] (order not guaranteed) # Set operations a = {1, 2, 3, 4} b = {3, 4, 5, 6} a | b # union → {1, 2, 3, 4, 5, 6} a & b # intersection → {3, 4} a - b # difference → {1, 2} a ^ b # symmetric diff → {1, 2, 5, 6}
Use {} only for non-empty sets; an empty {} creates a dict, not a set. Always write set() for an empty set.
Use .discard() over .remove() when the item may or may not be present — it is the safer default.
A frozenset is an immutable set — it can be used as a dictionary key or stored inside another set.

Choosing the right collection

Collection Ordered? Mutable? Duplicates? Best for
list ✅ Yes ✅ Yes ✅ Yes Sequences you build, sort, or grow over time
tuple ✅ Yes ❌ No ✅ Yes Fixed records, multi-value returns, dict keys
dict ✅ Yes (3.7+) ✅ Yes Keys: ❌ Key-based lookup, structured data, counting
set ❌ No ✅ Yes ❌ No Deduplication, fast membership, set algebra

Practice Questions
Typical interview and self-check questions on data structures
Mental model
Each collection solves a different problem. Ask yourself: does order matter? do I need to change it? are duplicates meaningful? do I need to look things up by name? Those four questions map directly onto the four types.
The Answer
Python's four core collections are: list — ordered, mutable, allows duplicates; use when you need a sequence you can grow or sort. tuple — ordered, immutable, allows duplicates; use for fixed-structure data, multi-value returns, or dict keys. dict — key-value mapping, O(1) lookup; use when items have meaningful names or when you need fast key-based access. set — unordered, unique items, O(1) membership; use for deduplication or set algebra.
my_list = [1, 2, 2, 3] # ordered, mutable, duplicates OK my_tuple = (1, 2, 2, 3) # ordered, immutable my_dict = {"a": 1, "b": 2} # key → value lookup my_set = {1, 2, 3} # unique items only
In practice: if you're unsure, start with a list. Upgrade to a dict when you catch yourself searching a list for a matching field, and to a set when you need to eliminate duplicates or test membership at scale.
{} alone creates an empty dict, not a set — use set() for an empty set.
Mental model
Think of indices as markers between elements, not on them. Slicing cuts between two markers. Negative indices count backward from the right: -1 is always the last element, regardless of list length.
The Answer
Slicing syntax is lst[start:stop:step]. The stop index is exclusive. Omitting start defaults to 0; omitting stop defaults to the end. Negative indices wrap around from the end: -1 is the last element, -2 is second-to-last. A negative step reverses the direction.
lst = [10, 20, 30, 40, 50] lst[1:4] # → [20, 30, 40] stop exclusive lst[:3] # → [10, 20, 30] from start lst[2:] # → [30, 40, 50] to end lst[-1] # → 50 last element lst[-3:] # → [30, 40, 50] last 3 lst[::-1] # → [50, 40, 30, 20, 10] reversed copy lst[:] # → [10, 20, 30, 40, 50] shallow copy
⚠️ Slicing never raises an IndexError — out-of-range bounds are silently clamped. But direct indexing like lst[10] will raise one.
lst[:] creates a shallow copy of the list — useful when you need to iterate and mutate separately.
Slicing works on any sequence type: strings, tuples, and ranges support the same [start:stop:step] syntax.
Mental model
Bracket access d[key] is like demanding a file from a filing cabinet — if it's not there, the cabinet throws an error. .get() is politely asking — if it's not there, you get a quiet None instead of a crash.
The Answer
Use dict.get(key) to return None when the key is absent, or dict.get(key, default) to supply your own fallback. The bare dict[key] syntax raises a KeyError if the key does not exist. You can also guard with if key in d: before accessing, or use a try/except KeyError block when a missing key is genuinely exceptional.
config = {"host": "localhost", "port": 5432} config["host"] # → "localhost" config["password"] # ❌ KeyError config.get("password") # → None (safe) config.get("password", "") # → "" (safe with fallback) # Pattern: return early on missing key host = config.get("host") or "127.0.0.1"
Use .get() as the default for reading dict values when absence is a normal, expected state.
Use collections.defaultdict when every missing key should automatically get the same starting value: from collections import defaultdict; counts = defaultdict(int).
Mental model
A dictionary finds values by computing a hash of the key — a fingerprint. A list's fingerprint would need to change every time the list changes, which would lose the value that was stored under it. Python forbids mutable keys to prevent this.
The Answer
A dictionary key must be hashable — it must have a stable __hash__ value that never changes during its lifetime, and it must support equality comparison. Immutable types — int, float, str, bool, tuple (of hashable items), and frozenset — are hashable and valid keys. list, dict, and set are mutable: their hash would need to change when their contents change, which would corrupt the dictionary's internal structure, so Python raises a TypeError if you try.
d = {} d[(1, 2)] = "tuple key — OK" d[frozenset({1, 2})] = "frozenset key — OK" d[[1, 2]] = "list key" # ❌ TypeError: unhashable type: 'list'
If you need a list as a key, convert it to a tuple first: d[tuple(my_list)] = value.
Custom classes are hashable by default (using their id as hash) unless they define __eq__ without __hash__.
Mental model
A for loop with append() is like a factory worker adding parts one by one with a separate motion for each. A list comprehension is the same factory but in a single automated pass — faster and more declarative.
The Answer
A list comprehension is a compact, single-expression syntax for creating a list: [expr for item in iterable if condition]. It is typically 20–30% faster than an equivalent loop with append() because the list is pre-allocated internally. More importantly, it reads as a declaration of what you want rather than how to build it.
# for loop with append — imperative squares = [] for x in range(1, 6): squares.append(x ** 2) # → [1, 4, 9, 16, 25] # list comprehension — declarative, faster squares = [x**2 for x in range(1, 6)] # with filter even_squares = [x**2 for x in range(1, 11) if x % 2 == 0] # → [4, 16, 36, 64, 100] # dict and set comprehensions work the same way square_map = {x: x**2 for x in range(5)} # dict unique_lens = {len(w) for w in ["hi", "hello", "hey"]} # set
⚠️ Keep comprehensions to one logical idea. If you need nested logic, side effects, or multiple statements, a plain for loop is clearer than a deeply nested comprehension.
Generator expressions ((expr for item in iterable) with round brackets) produce values lazily — use them when you only need to iterate once and want to save memory.
Dict and set comprehensions share the same syntax with {} instead of [].
Mental model
A shallow copy duplicates the outer container but all the inner objects are still shared. A deep copy duplicates everything, all the way down. For flat lists of numbers or strings, shallow is always fine. For nested structures, it depends on whether you need independence.
The Answer
A shallow copy creates a new container but the elements inside are still references to the same objects as the original. Mutating a nested mutable object through one copy affects the other. A deep copy (via copy.deepcopy()) recursively duplicates every nested object so the two copies are fully independent.
import copy original = [[1, 2], [3, 4]] shallow = original[:] # or list(original) or original.copy() deep = copy.deepcopy(original) original[0].append(99) print(shallow) # → [[1, 2, 99], [3, 4]] ← inner list is shared! print(deep) # → [[1, 2], [3, 4]] ← fully independent
For flat lists ([1, 2, 3]) shallow copy is always safe — integers and strings are immutable, so sharing references has no side effects.
dict.copy() and set.copy() are also shallow. Use copy.deepcopy() for any nested structure where independence matters.

Listen
Geordnete, veränderliche Sequenzen — Pythons vielseitigste Sammlung

Listen erstellen, indizieren und slicen

Eine Liste ist eine geordnete, veränderliche Sequenz, die Elemente beliebigen Typs halten kann, auch gemischte. Sie wird mit eckigen Klammern definiert. Der Zugriff erfolgt per nullbasiertem Index; negative Indizes zählen vom Ende.

obst = ["Apfel", "Banane", "Kirsche", "Dattel"] # Indizierung obst[0] # → "Apfel" (erstes Element) obst[-1] # → "Dattel" (letztes Element) obst[-2] # → "Kirsche" (zweitletztes) # Slicing [start : stop : schritt] — stop ist exklusiv obst[1:3] # → ["Banane", "Kirsche"] obst[:2] # → ["Apfel", "Banane"] obst[::2] # → ["Apfel", "Kirsche"] (jedes 2.) obst[::-1] # → ["Dattel", "Kirsche", "Banane", "Apfel"] (umgekehrt)

Listen verändern

zahlen = [3, 1, 4, 1, 5] zahlen.append(9) # ans Ende → [3, 1, 4, 1, 5, 9] zahlen.insert(0, 0) # an Index → [0, 3, 1, 4, 1, 5, 9] zahlen.remove(1) # entfernt ERSTE Vorkommen von 1 zahlen.pop() # entfernt & gibt letztes Element zurück zahlen.pop(0) # entfernt & gibt Element bei Index 0 zurück zahlen.sort() # sortiert in-place (gibt None zurück) zahlen.reverse() # kehrt in-place um (gibt None zurück) # Nicht-mutierende Alternativen sorted(zahlen) # neue sortierte Liste, zahlen unverändert zahlen + [10, 11] # Verkettung — neue Liste
append() ist O(1); insert(0, x) ist O(n) — Einfügen am Anfang verschiebt jedes Element. Für schnelle Front-Einfügungen collections.deque verwenden.
sort() und sorted() akzeptieren eine key-Funktion: sorted(woerter, key=str.lower).
Der in-Operator prüft Zugehörigkeit: "Apfel" in obst → True. Bei einer Liste ist das O(n) — ein Set verwenden für häufige Mitgliedschaftstests.

List Comprehensions

Eine List Comprehension erstellt eine neue Liste, indem ein Ausdruck auf jedes Element eines Iterables angewendet wird, optional gefiltert durch eine Bedingung. Sie ist schneller und lesbarer als eine äquivalente for-Schleife mit append().

# [ausdruck for element in iterable if bedingung] quadrate = [x**2 for x in range(1, 6)] # → [1, 4, 9, 16, 25] gerade = [x for x in range(10) if x % 2 == 0] # → [0, 2, 4, 6, 8] gross = [s.upper() for s in ["a", "b", "c"]] # → ["A", "B", "C"] # Verschachtelte Comprehension — 2D-Liste glätten matrix = [[1, 2], [3, 4], [5, 6]] flach = [n for zeile in matrix for n in zeile] # → [1, 2, 3, 4, 5, 6]

Tupel
Geordnete, unveränderliche Sequenzen — für feste Datensätze und sichere Rückgabewerte

Wann man ein Tupel statt einer Liste verwendet

Ein Tupel sieht aus wie eine Liste, verwendet aber runde Klammern und ist unveränderlich — sein Inhalt kann nach der Erstellung nicht mehr geändert werden. Das macht Tupel nützlich für Daten, die nicht verändert werden sollten, als Dictionary-Schlüssel und als leichtgewichtige Datensätze mit einer festen Anzahl von Feldern.

punkt = (3, 7) # 2D-Koordinate — semantisch fest rgb = (255, 128, 0) # Farbkanal — drei klar definierte Rollen # Indizierung funktioniert wie bei Listen punkt[0] # → 3 # Entpacken — jedes Element einem Namen zuweisen x, y = punkt lat, lon = (52.52, 13.40) # Variablen tauschen ohne Hilfsvariable a, b = 1, 2 a, b = b, a # → a=2, b=1 # Einelementiges Tupel braucht ein abschließendes Komma singleton = (42,) # NICHT (42) — das ist nur eine geklammerte int # Tupel als Dictionary-Schlüssel (Listen können keine Schlüssel sein) gitter = {(0, 0): "Start", (3, 3): "Ziel"}
Tupel sind etwas speichereffizienter und schneller zu iterieren als Listen — bevorzuge sie für von Natur aus feste Daten.
Funktionen, die mehrere Werte zurückgeben, geben tatsächlich ein Tupel zurück: return x, y ist return (x, y).
Named Tuples (collections.namedtuple) fügen Feldnamen zu Tupeln hinzu — selbstdokumentierend ohne den Overhead einer vollständigen Klasse.
🗂️
Dictionaries
Schlüssel-Wert-Zuordnungen mit O(1)-Durchschnittszugriff

Mit Dictionaries arbeiten

Ein Dictionary ordnet unveränderliche Schlüssel Werten beliebigen Typs zu. Suchen, Einfügen und Löschen sind durchschnittlich O(1). Seit Python 3.7 behalten Dictionaries die Einfügereihenfolge bei.

nutzer = {"name": "Alice", "alter": 30, "rolle": "admin"} # Lesen — zwei Wege nutzer["name"] # → "Alice" (KeyError wenn fehlt) nutzer.get("email") # → None (sicher, kein Fehler) nutzer.get("email", "n/a") # → "n/a" (eigener Standard) # Schreiben / aktualisieren nutzer["alter"] = 31 # vorhandenen Schlüssel aktualisieren nutzer["stadt"] = "Berlin" # neuen Schlüssel hinzufügen nutzer.update({"rolle": "owner", "stufe": 5}) # Massenaktualisierung # Löschen del nutzer["stadt"] # KeyError wenn nicht vorhanden nutzer.pop("stufe") # entfernt & gibt Wert zurück nutzer.pop("stufe", None) # sicheres pop — kein Fehler wenn fehlt # Iterieren for schluessel in nutzer: # Schlüssel print(schluessel, nutzer[schluessel]) for schluessel, wert in nutzer.items(): # Schlüssel-Wert-Paare for wert in nutzer.values(): # nur Werte
Immer .get(schluessel, standard) gegenüber [schluessel] bevorzugen, wenn der Schlüssel möglicherweise fehlt.
collections.defaultdict erstellt automatisch einen Standardwert für fehlende Schlüssel — nützlich zum Gruppieren oder Zählen.
Dict Comprehensions folgen dem gleichen Muster: {k: v for k, v in paare}.

Gültige Dictionary-Schlüssel

Nur hashbare (unveränderliche) Objekte können Dictionary-Schlüssel sein — Integer, Float, Strings, Booleans und Tupel aus hashbaren Elementen. Listen, Dicts und Sets sind veränderlich und daher keine gültigen Schlüssel.

gueltig = {1: "int", 3.14: "float", "x": "str", (0,0): "tuple"} {[1, 2]: "liste"} # ❌ TypeError: unhashable type: 'list'
🔵
Sets (Mengen)
Ungeordnete Sammlungen einzigartiger, hashbarer Elemente

Sets für Deduplizierung und Mitgliedschaftstests

Ein Set speichert nur einzigartige Elemente ohne definierte Reihenfolge. Wie Dictionary-Schlüssel müssen Set-Elemente hashbar sein. Mitgliedschaftstests sind O(1) — wesentlich schneller als das Durchsuchen einer Liste.

farben = {"rot", "grün", "blau", "rot"} # Duplikat entfernt print(farben) # → {'rot', 'grün', 'blau'} (Reihenfolge variiert) # Mitgliedschaft — O(1) vs. O(n) der Liste "rot" in farben # → True # Verändern farben.add("gelb") farben.discard("grün") # sicher — kein Fehler wenn nicht vorhanden farben.remove("grün") # KeyError wenn nicht vorhanden # Deduplizierung zahlen = [1, 2, 2, 3, 3, 3] eindeutig = list(set(zahlen)) # → [1, 2, 3] (Reihenfolge nicht garantiert) # Mengenoperationen a = {1, 2, 3, 4} b = {3, 4, 5, 6} a | b # Vereinigung → {1, 2, 3, 4, 5, 6} a & b # Schnittmenge → {3, 4} a - b # Differenz → {1, 2} a ^ b # Sym. Differenz → {1, 2, 5, 6}
{} allein erstellt ein leeres Dict, kein Set — für ein leeres Set immer set() schreiben.
.discard() gegenüber .remove() bevorzugen, wenn das Element möglicherweise nicht vorhanden ist.
Ein frozenset ist ein unveränderliches Set — kann als Dictionary-Schlüssel verwendet oder in einem anderen Set gespeichert werden.

Die richtige Sammlung wählen

Sammlung Geordnet? Veränderlich? Duplikate? Am besten für
list ✅ Ja ✅ Ja ✅ Ja Sequenzen, die wachsen, sortiert oder verändert werden
tuple ✅ Ja ❌ Nein ✅ Ja Feste Datensätze, Mehrfach-Rückgaben, Dict-Schlüssel
dict ✅ Ja (3.7+) ✅ Ja Schlüssel: ❌ Schlüsselbasierte Suche, strukturierte Daten, Zählen
set ❌ Nein ✅ Ja ❌ Nein Deduplizierung, schnelle Mitgliedschaft, Mengenalgebra

Übungsfragen
Typische Interview- und Selbstkontrollfragen zu Datenstrukturen
Mentales Modell
Jede Sammlung löst ein anderes Problem. Frag dich: Ist Reihenfolge wichtig? Muss ich sie ändern? Sind Duplikate bedeutsam? Muss ich Dinge beim Namen suchen? Diese vier Fragen entsprechen direkt den vier Typen.
Die Antwort
Pythons vier Kernsammlungen: list — geordnet, veränderlich, Duplikate erlaubt; wenn man eine wachsende oder zu sortierende Sequenz braucht. tuple — geordnet, unveränderlich; für Festdaten, Mehrfach-Rückgaben oder Dict-Schlüssel. dict — Schlüssel-Wert-Zuordnung, O(1)-Suche; wenn Elemente sinnvolle Namen haben oder schnellen Schlüsselzugriff benötigen. set — ungeordnet, einzigartige Elemente, O(1)-Mitgliedschaft; für Deduplizierung oder Mengenalgebra.
meine_liste = [1, 2, 2, 3] # geordnet, veränderlich, Duplikate OK mein_tupel = (1, 2, 2, 3) # geordnet, unveränderlich mein_dict = {"a": 1, "b": 2} # Schlüssel → Wert-Suche mein_set = {1, 2, 3} # nur einzigartige Elemente
Wenn unsicher: mit einer Liste anfangen. Zu einem Dict wechseln, wenn man eine Liste nach einem Feld durchsucht; zu einem Set, wenn man Duplikate entfernen oder Mitgliedschaft effizient prüfen muss.
{} allein erstellt ein leeres Dict, kein Set — für ein leeres Set set() schreiben.
Mentales Modell
Stell dir Indizes als Markierungen zwischen den Elementen vor, nicht auf ihnen. Slicing schneidet zwischen zwei Markierungen. Negative Indizes zählen rückwärts vom rechten Ende: -1 ist immer das letzte Element.
Die Antwort
Slicing-Syntax: lst[start:stop:schritt]. Der stop-Index ist exklusiv. Fehlt start, beginnt es bei 0; fehlt stop, geht es bis zum Ende. Negative Indizes zählen vom Ende: -1 ist das letzte Element. Ein negativer Schritt kehrt die Richtung um.
lst = [10, 20, 30, 40, 50] lst[1:4] # → [20, 30, 40] stop exklusiv lst[:3] # → [10, 20, 30] vom Anfang lst[2:] # → [30, 40, 50] bis zum Ende lst[-1] # → 50 letztes Element lst[-3:] # → [30, 40, 50] letzte 3 lst[::-1] # → [50, 40, 30, 20, 10] umgekehrte Kopie lst[:] # → [10, 20, 30, 40, 50] flache Kopie
⚠️ Slicing löst nie einen IndexError aus — außerhalb liegende Grenzen werden still begrenzt. Direkter Zugriff wie lst[10] hingegen schon.
lst[:] erstellt eine flache Kopie der Liste — nützlich, wenn man gleichzeitig iterieren und verändern möchte.
Slicing funktioniert auf allen Sequenztypen: Strings, Tupel und Ranges unterstützen dieselbe [start:stop:schritt]-Syntax.
Mentales Modell
Der Klammerzugriff d[schlüssel] ist wie das Verlangen nach einer Akte — wenn sie nicht da ist, wirft der Schrank einen Fehler. .get() ist höflich nachfragen — wenn sie nicht da ist, bekommst du leise None statt eines Absturzes.
Die Antwort
dict.get(schluessel) gibt None zurück, wenn der Schlüssel fehlt; dict.get(schluessel, standard) liefert einen eigenen Fallback. Der bloße dict[schluessel]-Zugriff löst einen KeyError aus, wenn der Schlüssel nicht vorhanden ist.
konfig = {"host": "localhost", "port": 5432} konfig["host"] # → "localhost" konfig["passwort"] # ❌ KeyError konfig.get("passwort") # → None (sicher) konfig.get("passwort", "") # → "" (sicher mit Fallback) # Muster: bei fehlendem Schlüssel früh zurückkehren host = konfig.get("host") or "127.0.0.1"
.get() als Standard verwenden, wenn ein fehlendes Schlüssel ein normaler Zustand ist.
collections.defaultdict verwenden, wenn jeder fehlende Schlüssel automatisch denselben Startwert bekommen soll: defaultdict(int) für Zähler.
Mentales Modell
Ein Dictionary findet Werte durch Berechnung eines Hashes des Schlüssels — ein Fingerabdruck. Der Fingerabdruck einer Liste müsste sich jedes Mal ändern, wenn die Liste geändert wird, was den gespeicherten Wert "verlieren" würde. Python verbietet veränderliche Schlüssel, um dies zu verhindern.
Die Antwort
Ein Dictionary-Schlüssel muss hashbar sein — er muss einen stabilen __hash__-Wert haben, der sich nie ändert. Unveränderliche Typen — int, float, str, bool, tuple (aus hashbaren Elementen) und frozenset — sind hashbar. list, dict und set sind veränderlich und würden die interne Struktur des Dicts beschädigen — Python löst einen TypeError aus.
d = {} d[(1, 2)] = "Tupel-Schlüssel — OK" d[frozenset({1, 2})] = "frozenset — OK" d[[1, 2]] = "listen-schluessel" # ❌ TypeError: unhashable type: 'list'
Wenn eine Liste als Schlüssel benötigt wird, zuerst in ein Tupel umwandeln: d[tuple(meine_liste)] = wert.
Eigene Klassen sind standardmäßig hashbar (nutzen ihre id als Hash), außer sie definieren __eq__ ohne __hash__.
Mentales Modell
Eine for-Schleife mit append() ist wie ein Fabrikarbeiter, der Teile einzeln hinzufügt. Eine List Comprehension ist dieselbe Fabrik in einem einzigen automatisierten Durchgang — schneller und deklarativer.
Die Antwort
Eine List Comprehension ist eine kompakte, einzeilige Syntax zum Erstellen einer Liste: [ausdruck for element in iterable if bedingung]. Sie ist typischerweise 20–30 % schneller als eine äquivalente Schleife mit append(), weil die Liste intern vorab allokiert wird. Vor allem aber liest sie sich als Aussage über das Was statt über das Wie.
# for-Schleife mit append — imperativ quadrate = [] for x in range(1, 6): quadrate.append(x ** 2) # → [1, 4, 9, 16, 25] # List Comprehension — deklarativ, schneller quadrate = [x**2 for x in range(1, 6)] # mit Filter gerade_quadrate = [x**2 for x in range(1, 11) if x % 2 == 0] # → [4, 16, 36, 64, 100] # Dict- und Set-Comprehensions folgen demselben Muster quad_map = {x: x**2 for x in range(5)} # dict laengen = {len(w) for w in ["hi", "hallo", "hey"]} # set
⚠️ Comprehensions auf eine logische Idee beschränken. Bei verschachtelter Logik, Seiteneffekten oder mehreren Anweisungen ist eine normale for-Schleife klarer.
Generator-Ausdrücke ((ausdruck for element in iterable) mit runden Klammern) liefern Werte lazy — verwenden, wenn nur einmal iteriert wird und Speicher gespart werden soll.
Dict- und Set-Comprehensions verwenden dieselbe Syntax mit {} statt [].
Mentales Modell
Eine flache Kopie dupliziert nur den äußeren Container — alle inneren Objekte werden noch geteilt. Eine tiefe Kopie dupliziert alles bis in die letzte Ebene. Bei flachen Listen aus Zahlen oder Strings ist flach immer ausreichend. Bei verschachtelten Strukturen kommt es darauf an, ob man Unabhängigkeit braucht.
Die Antwort
Eine flache Kopie erstellt einen neuen Container, aber die darin enthaltenen Elemente sind noch Referenzen auf dieselben Objekte wie im Original. Mutiert man ein verschachteltes veränderliches Objekt durch eine Kopie, betrifft das auch die andere. Eine tiefe Kopie (via copy.deepcopy()) dupliziert rekursiv jedes verschachtelte Objekt, sodass beide Kopien vollständig unabhängig sind.
import copy original = [[1, 2], [3, 4]] flach = original[:] # oder list(original) oder original.copy() tief = copy.deepcopy(original) original[0].append(99) print(flach) # → [[1, 2, 99], [3, 4]] ← innere Liste geteilt! print(tief) # → [[1, 2], [3, 4]] ← vollständig unabhängig
Für flache Listen ([1, 2, 3]) ist eine flache Kopie immer sicher — Integer und Strings sind unveränderlich.
dict.copy() und set.copy() sind ebenfalls flach. Für verschachtelte Strukturen copy.deepcopy() verwenden.
